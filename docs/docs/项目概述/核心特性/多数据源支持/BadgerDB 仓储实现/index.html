简介​

Sparrow 框架中的 BadgerDB 仓储实现是一个基于 Dgraph Labs 开源的 BadgerDB 键值存储数据库的完整数据访问层实现。该实现提供了对泛型实体的完整 CRUD 操作支持，包括单实体操作、批量操作、分页查询、条件查询等高级功能。

BadgerDB 是一个高性能的嵌入式 NoSQL 数据库，采用 LSM-Tree（Log-Structured Merge-Tree）存储引擎，专为现代硬件特性进行了优化。它提供了 ACID 事务、MVCC（多版本并发控制）、自动压缩、快照等功能，特别适合需要高并发写入和低延迟访问的应用场景。

使用示例​

基本 CRUD 操作​

// 创建任务实体
task := entity.NewTask("email", map[string]interface{}{"to": "user@example.com"})

// 保存实体
err := repo.Save(context.Background(), task)

// 查找实体
foundTask, err := repo.FindByID(context.Background(), task.Id)

// 更新实体
task.Status = "sent"
err = repo.Update(context.Background(), task)

// 删除实体
err = repo.Delete(context.Background(), task.Id)

// 条件查询

性能优化建议​

  1. 合理配置内存：根据工作负载调整 MemTableSize 和 NumMemtables
  2. 监控压缩效果：定期检查 SSTable 文件大小和压缩比
  3. 优化查询模式：尽量使用 ID 查询而非全表扫描
  4. 批量处理：使用 SaveBatch 和 DeleteBatch 提升吞吐量
