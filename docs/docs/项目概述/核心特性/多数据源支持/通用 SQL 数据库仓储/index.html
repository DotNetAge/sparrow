简介​

本文件面向 Sparrow 框架的通用 SQL 数据库仓储实现，系统性阐述 SqlDBRepository 的设计架构与数据库抽象层，涵盖以下要点：

  • 数据库驱动抽象与统一接口：通过 database/sql 抽象层屏蔽底层驱动差异，支持 SQLite、MySQL、PostgreSQL 等。
  • SQL 语句生成与参数绑定：基于反射自动映射实体字段，生成 INSERT/UPDATE/SELECT/DELETE 等语句，并采用占位符参数绑定，确保安全性与可维护性。
  • 软删除与值对象支持：内置软删除检测与处理，支持嵌入式值对象的字段提取与映射。
  • 动态查询构建：提供多条件查询、排序、分页与统计能力，统一查询选项模型。
  • 配置与连接管理：SQL 配置结构化，支持 DSN 构造与事件存储专用 DSN。
  • 错误处理与跨数据库兼容策略：统一错误类型与消息，结合数据库方言差异制定兼容策略。

项目结构​

仓库中与 SQL 仓储相关的核心文件分布如下：

  • 通用 SQL 仓储：pkg/persistence/repo/sqldb.go
  • SQL 仓储测试：pkg/persistence/repo/sqldb_test.go
  • PostgreSQL 仓储：pkg/persistence/repo/postgres.go
  • 内存仓储（对比参考）：pkg/persistence/repo/memory.go
  • 数据库配置：pkg/config/sql.go
  • 启动阶段数据库聚合：pkg/bootstrap/db.go
  • 实体基类与接口：pkg/entity/base_entity.go、pkg/entity/entity.go
  • 用例层仓储接口与查询选项：pkg/usecase/base_repo.go
  • 仓储错误类型：pkg/errs/repo.go
  • SqlDBRepository：基于 database/sql 的泛型仓储，负责将实体映射到关系型数据库表，提供 CRUD、批量操作、分页、条件查询、统计与随机查询等能力。
  • PostgresRepository：基于 sqlx 的 PostgreSQL 仓储，提供更丰富的类型支持与事务内批量操作。
  • MemoryRepository：内存仓储，用于测试与无持久化场景，便于对比与验证。
  • SQLConfig：统一的 SQL 配置结构，提供 DSN 与事件存储 DSN 的构造方法。
  • Database：应用启动阶段聚合的数据库客户端（Redis、SQL、Badger），便于统一管理。
  • Entity/BaseEntity：实体接口与基础实体，提供 ID、创建/更新时间等标准字段。
  • BaseRepository[T]/QueryOptions：用例层仓储接口与查询条件、排序、分页选项的抽象。

通用 SQL 仓储通过以下层次实现：

  • 抽象层：usecase.BaseRepository[T] 定义仓储接口与默认行为，具体仓储实现覆盖核心方法。
  • 映射层：反射提取实体字段，支持嵌入结构体与值对象，自动完成列名转换（驼峰/蛇形）。
  • 语句层：动态生成 SQL，使用占位符参数绑定，避免拼接风险。
  • 执行层：统一使用 database/sql 或 sqlx 执行，支持事务与批量操作。
  • 配置层：SQLConfig 提供 DSN 构造，支持不同数据库驱动。
  • 错误层：RepositoryError 统一错误类型，便于上层处理。

SqlDBRepository 设计与实现​

  • 泛型与继承：继承 usecase.BaseRepository[T]，组合 *sql.DB，通过反射实现字段映射与 SQL 生成。
  • 表名推断：基于实体类型名推断表名（去除包路径与指针，转小写）。
  • 软删除检测：通过反射检测实体是否包含 DeletedAt 字段，自动在查询与删除时加入软删除逻辑。
  • 事务与批量：SaveBatch 与批量查询均在事务中执行，保证一致性。
  • 字段映射：toSnakeCase/toCamelCase 实现驼峰与蛇形命名互转；extractFields 支持嵌入结构体与值对象字段提取。
  • 参数绑定：占位符 "?" 与反射收集的值列表，避免 SQL 注入。
  • 查询构建：buildCondition 支持 EQ、NEQ、GT、GTE、LT、LTE、LIKE、IN、IS_NULL、IS_NOT_NULL 等操作符。
  • 扫描机制：scanRow/scanEntity 结合 findFieldForColumn 与 nullableTime 处理 time.Time 的 NULL 场景。

  • 基于 sqlx：提供更丰富的类型支持与事务内批量操作。

  • 参数占位符：使用 "$n" 形式的参数占位符，与通用仓储的 "?" 不同。

  • 扫描与类型：scanEntity 使用 sqlx.Rows，支持复杂类型与数组类型处理。

  • 条件构建：buildCondition 支持 LIKE（ILIKE）、IN（ANY）等 PostgreSQL 特性。

  • 事务内操作：existsInTx、insertInTx、updateInTx 提供事务内一致性保障。

  • 并发安全：使用读写锁保护实体映射。

  • 查询实现：支持按字段查找、条件查询、分页、随机抽取等。

  • 类型比较：matchCondition 支持数值、字符串、布尔与时间类型的比较。

  • SQLConfig：包含 Driver、Host、Port、User、Password、Dbname、ESDbname 等字段，提供 DSN 与事件存储 DSN 构造方法。

  • Database：应用启动阶段聚合 Redis、SQL、Badger 客户端，便于统一管理与注入。

  • Entity 接口：统一的实体约束，要求实现 ID、创建/更新时间的 getter/setter。

  • BaseEntity：提供标准字段与构造函数，便于实体快速实现。

  • BaseRepository[T]：定义仓储接口与默认错误处理，QueryOptions/QueryCondition/SortField 提供统一查询模型。

  • SqlDBRepository 依赖 usecase.BaseRepository[T]、entity.Entity、errs.RepositoryError、database/sql。

  • 字段映射与扫描依赖 reflect、strings、time、unicode。

  • PostgreSQL 仓储依赖 sqlx、logger、errs。

  • 内存仓储依赖 sync、reflect、math/rand、time。

  • 配置依赖 config.SQLConfig。

  • 启动阶段依赖 bootstrap.Database。

  • 批量操作：SaveBatch 与 DeleteBatch 在事务中执行，减少往返次数，提升吞吐。
  • 参数绑定：使用占位符与反射收集值，避免字符串拼接带来的性能与安全问题。
  • 字段映射：toSnakeCase/toCamelCase 与 extractFields 在运行时进行，建议在高频路径中缓存列名映射或在实体层面减少复杂嵌套。
  • 查询优化：FindWithConditions 自动拼接 WHERE/ORDER/LIMIT，建议配合索引与合适的分页参数。
  • 时间处理：nullableTime 适配 time.Time 的 NULL 场景，避免额外的类型转换开销。
  • PostgreSQL 差异：PostgreSQL 使用 "$n" 占位符与 sqlx 的类型支持，适合复杂类型与数组字段。

故障排查指南​

  • 未找到实体：RepositoryError 提供明确的 EntityType、Operation、ID、Message 信息，便于定位问题。
  • ID 为空：Save/Update/Delete 前检查实体 ID，避免非法操作。
  • 查询异常：scanRow/scanEntity 中的字段映射失败通常由列名与实体字段不匹配导致，需确认命名约定与标签。
  • 软删除：若实体包含 DeletedAt 字段，查询与删除会自动过滤软删除记录，注意业务逻辑对软删除的处理。
  • PostgreSQL IN/ANY：PostgreSQL 的 IN 条件使用 ANY($n)，确保传入切片类型与长度正确。

SqlDBRepository 通过 database/sql 抽象层实现了对多种 SQL 数据库的统一访问，结合反射映射与动态 SQL 生成，提供了灵活、安全且可扩展的仓储能力。其软删除、值对象、批量操作与条件查询等功能，满足大多数业务场景需求。配合 SQLConfig 与启动阶段的 Database 聚合，可实现配置驱动的数据库接入与管理。对于 PostgreSQL 等高级特性，PostgresRepository 提供了更完善的类型支持与事务内批量操作能力，可作为高性能场景的首选。

使用示例（概念性说明）​

  • 创建数据库连接：使用 SQLConfig.Dsn() 构造 DSN，连接 SQLite/MySQL/PostgreSQL。
  • 初始化仓储：NewSqlDBRepository*YourEntity 或 NewPostgresRepository[*YourEntity](dbx, tableName, logger)。
  • 基本操作：Save/FindById/FindAll/Update/Delete/Exists。
  • 批量操作：SaveBatch/FindByIDs/DeleteBatch。
  • 条件查询：FindWithConditions/CountWithConditions，结合 QueryOptions/QueryCondition/SortField。
  • 随机查询：Random(take)。
