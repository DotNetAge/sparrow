简介​

WebSocket 适配器是 Sparrow 项目中的实时通信基础设施，提供了完整的 WebSocket 连接管理、消息处理和实时通信机制。该适配器采用简洁高效的架构设计，支持全局广播、单点发送、用户广播等多种消息传输模式，并内置了完善的连接池管理和资源清理策略。

该适配器的核心特性包括：

  • 基于 Gorilla WebSocket 库的高性能连接管理
  • 线程安全的客户端连接池
  • 多种消息传输模式（文本、JSON、二进制）
  • 自定义跨域校验和心跳检测机制
  • 完善的错误处理和自动重连机制
  • 与 HTTP 适配器的无缝集成

连接池优化​

WebSocket 适配器采用了以下性能优化策略：

  1. 线程安全设计：使用 RWMutex 实现读写分离，提高并发性能
  2. 延迟初始化：连接池仅在有客户端连接时才创建
  3. 批量操作：广播操作使用批量发送减少系统调用次数
  4. 内存管理：及时清理断开连接的客户端资源

心跳机制​

心跳检测机制的性能考量：

  • 默认 10 秒的心跳间隔平衡了连接保持和资源消耗
  • 使用定时器而非轮询，减少 CPU 占用
  • 异常情况下快速清理失效连接

常见问题诊断​

  1. 连接升级失败

    • 检查 WebSocket 握手头部是否正确设置
    • 验证跨域校验规则配置
    • 查看网络连接状态

  2. 消息发送失败

    • 确认客户端是否仍然在线
    • 检查连接池中是否存在该客户端
    • 验证消息格式和大小限制

  3. 心跳检测异常

    • 检查网络延迟和防火墙设置
    • 验证心跳间隔配置是否合理
    • 查看连接池清理日志

日志分析​

WebSocket 适配器的关键日志点：

  • [WebSocket] 连接升级失败：连接建立阶段的问题
  • [WebSocket] 客户端连接：新连接建立成功
  • [WebSocket] 客户端断开：连接断开事件
  • [WebSocket] 读取消息失败：消息读取异常
  • [WebSocket] 心跳失败：心跳检测异常

WebSocket 适配器为 Sparrow 项目提供了强大而灵活的实时通信能力。其设计特点包括：

  1. 简洁高效：核心功能集中在单一文件中，易于理解和维护
  2. 功能完整：支持多种消息传输模式和连接管理策略
  3. 性能优秀：采用线程安全设计和优化的资源管理
  4. 易于集成：与 Gin 框架和认证系统无缝集成
  5. 可扩展性强：提供配置选项和扩展点

API 使用示例​

基础连接管理​

// 在 Gin 路由中启用 WebSocket 中间件
r.Use(ws.Middleware())

// 自定义跨域校验
r.Use(ws.Middleware(
    ws.WithCheckOrigin(func(r *http.Request) bool {
        // 实现自定义跨域校验逻辑
        return true
    })
))

// 设置自定义心跳间隔
r.Use(ws.Middleware(
    ws.WithHeartbeatInterval(30 * time.Second)
))

消息发送示例​

// 全局广播
ws.BroadcastText("Hello WebSocket!")

// JSON 格式广播
stats := map[string]interface{}{
    "type": "system_stats",
    "online_count": ws.GetOnlineCount(),
    "server_time": time.Now().Format("2006-01-02 15:04:05"),
}
ws.BroadcastJSON(stats)

// 发送给指定客户端
ws.SendTextToClient("client_123", "Hello specific client!")

// 发送给指定用户
ws.SendTextToUser("user_456", "Hello user!")

会话管理集成​

// 会话处理器示例
func (h *SessionHandler) CreateSession(c *gin.Context) {
    id := uuid.New().String()
    session, err := h.sessionService.CreateSession(c.Request.Context(), id)
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{"error": err.Error()})
        return
    }
    
    // 创建会话后可以触发 WebSocket 通知
    ws.SendTextToUser(id, "Session created successfully!")
    
    c.JSON(http.StatusCreated, session)
}
