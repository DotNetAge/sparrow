简介​

本文件为事件投影系统的全面技术文档，面向数据架构师与后端工程师，系统阐述投影器的工作原理、事件到读模型的转换过程、索引构建机制，以及全量投影与增量投影的实现策略、投影状态管理与一致性保障。文档还展示了事件过滤器、投影处理器、索引更新机制的实现方式，涵盖并行处理、批量操作与事务管理的最佳实践，并提供性能优化、缓存策略与数据同步机制的建议。

项目结构​

事件投影系统位于 pkg/projection 与 pkg/adapter/tasks、pkg/tasks 等目录中，围绕“事件读取 → 聚合根索引 → 投影计算 → 视图存储”的主链路展开；同时通过任务调度器实现全量投影的自动化执行与并发控制。

  • 接口层（projection/core.go）
    • 聚合根索引器接口：提供按聚合类型获取有效聚合根ID的能力，用于全量/定时投影。
    • 事件读取器接口：按聚合类型与事件类型读取事件集合，用于全量投影的事件源。
    • 投影器接口：将事件流重放并投影为视图实体，供存储层写入。
  • 全量投影器（projection/full.go）
    • 组合事件读取器、索引器与投影器，按事件类型批量读取事件，再遍历聚合根ID逐一投影。
  • JetStream 实现
    • JetStream索引器：基于JetStream消费者扫描事件流，收集指定聚合类型的聚合根ID。
    • JetStream事件读取器：基于JetStream消费者扫描事件流，按聚合类型与事件类型筛选并去重最高版本事件。
  • 任务适配器（adapter/tasks/full_proj.go）
    • 全量投影任务：封装“索引聚合根 → 读取事件 → 投影 → 存储”的完整流程，支持日志与错误处理。
  • 任务调度器（tasks/*）
    • 任务与调度接口：统一任务生命周期、状态与重试策略。
    • 顺序调度器：严格串行执行，适合定时/周期任务。
    • 并发调度器：控制最大并发，支持任务超时、重试与过期清理。
  • 应用集成（bootstrap/*）
    • 注册投影器：通过容器注册JetStream事件读取器与索引器，便于任务调用。
  • 仓储接口（usecase/repo.go）
    • 提供泛型仓储能力，支持单条与批量持久化，用于视图写入。
  • 实体与事件接口（entity/*）
    • 定义领域事件与实体的最小契约，确保投影器与任务适配器的类型安全。

事件投影系统采用“事件驱动 + 任务编排”的架构：事件由NATS JetStream承载，索引器与事件读取器负责从流中抽取聚合根ID与事件；投影器将事件流重放为视图实体；任务调度器负责全量投影的触发、并发与重试；最终通过仓储接口写入读库。

组件A：全量投影器（projection/full.go）​

  • 职责
    • 以事件类型为维度批量读取事件；
    • 以聚合类型为维度枚举聚合根ID；
    • 对每个聚合根ID调用投影器生成视图并持久化。
  • 关键点
    • 事件读取：按事件类型过滤，内部去重保留最高版本事件；
    • 聚合根枚举：按聚合类型过滤，收集所有有效ID；
    • 投影执行：逐个聚合根重放事件流，生成视图实体；
    • 错误传播：任一步骤失败即终止，返回错误。
  • 性能特性
    • 顺序遍历聚合根ID，天然串行；
    • 事件读取器内部批处理与去重，降低重复事件影响。
  • 职责
    • 基于JetStream消费者扫描事件流；
    • 按聚合类型与事件类型过滤；
    • 分页拉取消息，去重保留最高版本事件；
    • 反序列化事件并确认消息。
  • 关键点
    • 过滤主题：按聚合类型构造过滤主题；
    • 消费策略：即时投递、显式确认；
    • 批处理：固定批次大小，分页消费；
    • 去重：按聚合根ID保留最高版本事件；
    • 错误处理：流不存在、消费者创建失败、反序列化失败均记录日志并返回。
  • 性能特性
    • 分页与批处理降低内存峰值；
    • 去重避免重复事件导致的重复投影。
  • 职责
    • 基于JetStream消费者扫描事件流；
    • 按聚合类型过滤；
    • 收集所有出现过的聚合根ID并去重。
  • 关键点
    • 过滤主题：按聚合类型构造过滤主题；
    • 消费策略：即时投递、显式确认；
    • 去重：使用map去重；
    • 错误处理：流不存在、消费者创建失败、反序列化失败均记录日志并返回。
  • 性能特性
    • 与事件读取器一致的批处理与去重策略。
  • 职责
    • 封装“索引聚合根 → 读取事件 → 投影 → 存储”的完整流程；
    • 提供日志记录与错误传播；
    • 无事件的聚合根跳过处理。
  • 关键点
    • 事件读取：使用消息层StreamReader按聚合根ID读取事件流；
    • 投影执行：调用投影器生成视图；
    • 视图存储：使用泛型仓储接口Save，建议使用UPSERT保证幂等；
    • 错误处理：任一步骤失败立即返回，避免部分更新。
  • 并发与批量
    • 该适配器本身不引入并发，可在调度器层面并行执行多个任务实例。
  • 任务与调度接口（tasks/core.go）
    • 统一任务生命周期：调度、取消、状态查询、列表、关闭；
    • 任务信息：状态、调度时间、重试次数、错误、TTL等；
    • 执行模式：顺序与并发两种模式。
  • 顺序调度器（sequential_scheduler.go）
    • 严格串行执行，适合定时/周期任务；
    • 内置指数退避重试与过期清理；
    • 任务队列与待执行队列配合，保证有序性。
  • 并发调度器（concurrent_scheduler.go）
    • 控制最大并发数，避免资源争用；
    • 轮询等待可用槽位，超时控制；
    • 支持任务超时、重试与过期清理；
    • 条件变量与互斥锁协调活跃任务数。
  • 应用启动（bootstrap/app.go）
    • 提供NATS连接、流发布器、流读取器工厂；
    • 提供优雅关闭与重试启动机制。
  • 投影注册（bootstrap/opts.go）
    • WithProjection：向容器注册全量投影器，注入JetStream事件读取器与索引器。

  • 领域事件接口（entity/domain_event.go）

    • 提供事件ID、类型、聚合ID、聚合类型、版本等；
    • 版本用于事件溯源与去重。

  • 实体接口（entity/entity.go）

    • 提供实体ID、创建/更新时间等；
    • 视图实体遵循此接口，便于统一存储。

  • 组件耦合

    • projection/full.go 依赖 EventReader、AggregateIndexer、Projector；
    • adapter/tasks/full_proj.go 依赖 Indexer、StreamReader、Projector、Repository；
    • tasks/* 与具体实现解耦，通过接口与容器注入；
    • bootstrap/opts.go 通过容器注册投影器，避免硬编码依赖。

  • 外部依赖

    • NATS JetStream：事件流存储与消费者；
    • 日志器：统一记录错误与运行状态；
    • 仓储接口：抽象视图持久化。
  • 事件读取与索引
    • 分页与批处理：事件读取器与索引器均采用固定批次大小分页拉取，降低内存峰值。
    • 去重策略：事件读取器按聚合根ID保留最高版本事件，避免重复投影。
  • 并发与吞吐
    • 顺序调度器：适合定时/周期任务，保证有序性与稳定性。
    • 并发调度器：通过最大并发数控制资源占用，结合超时与重试提升可靠性。
  • 批量与幂等
    • 仓储接口提供SaveBatch等批量能力，建议在视图存储层实现UPSERT以保证幂等。
  • 缓存策略
    • 可在投影器层缓存热点聚合根的视图，减少重复计算；
    • 在任务调度器层缓存最近任务状态，降低查询开销。
  • 数据同步
    • 事件读取器与索引器均基于JetStream消费者，具备高可靠与顺序保证；
    • 建议在视图存储层启用事务或批量提交，确保一致性。

故障排查指南​

  • 常见问题定位
    • NATS连接失败：检查NATS URL与认证配置，确认JetStream启用；
    • 流不存在：事件读取器与索引器在流不存在时记录警告并返回空结果，需先创建流；
    • 反序列化失败：事件负载格式异常，记录错误并跳过该消息；
    • 消费者创建失败：检查权限与流配置。
  • 日志与监控
    • 事件读取器与索引器均记录错误日志，便于定位问题；
    • 任务适配器记录聚合根处理进度与错误，便于追踪。
  • 重试与回滚
    • 顺序与并发调度器均支持重试与过期清理，建议合理设置最大重试次数与TTL；
    • 视图存储建议使用UPSERT，避免部分更新导致的数据不一致。

事件投影系统通过清晰的接口分层与任务编排，实现了从事件流到读模型的高效转换。JetStream作为事件基础设施提供了可靠的事件存储与消费者能力；任务调度器在不同执行模式下满足顺序性与吞吐性的需求；全量投影器与适配器则将复杂流程封装为可复用的组件。结合批量操作、幂等写入与合理的缓存策略，系统能够在高并发场景下保持稳定与高性能。
