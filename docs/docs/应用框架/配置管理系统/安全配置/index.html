简介​

本文件系统性梳理本项目的“安全配置”能力，重点覆盖以下方面：

  • Casbin 权限控制：模型文件路径、策略文件位置、权限规则执行流程
  • JWT 认证：密钥管理、过期时间、签名算法、Claims 结构与中间件校验
  • RBAC 权限模型：角色定义、权限分配、资源访问控制
  • RSA 客户端签名认证：请求头约定、时间窗口、随机数防重放、签名验证
  • 安全最佳实践：密码策略、会话管理、API 保护等建议

项目结构​

围绕安全配置的关键目录与文件如下：

  • 配置层：应用配置、Casbin 配置、配置加载与校验
  • 认证与授权：JWT 令牌生成与校验、RBAC 中间件、RSA 客户端签名中间件
  • 上下文与主体：UserClaims、Principal 的传递与使用
  • 引导层：Authorization 组合 TokenGenerator 与中间件注册
  • Casbin 权限控制
    • 配置结构：包含模型文件路径与策略文件路径
    • 中间件：从上下文提取用户名，结合请求 URI 与方法执行 Enforce
  • JWT 认证
    • 令牌结构：包含访问令牌、刷新令牌、作用域、过期时长
    • 生成器：支持按访问令牌与刷新令牌分别设置过期时间，使用 HS256 签名
    • 中间件：校验 Bearer 令牌、签名算法、过期与未生效、签名有效性；注入 UserClaims 与 Principal
  • RBAC 权限模型
    • 角色与权限：UserClaims 中包含角色数组，Casbin 中间件据此进行资源访问控制
  • RSA 客户端签名认证
    • 请求头约定：客户端ID、签名、随机数、时间戳
    • 防重放：NonceStore + 时间窗口
    • 签名验证：SHA256 + RSA-PSS

Casbin 权限控制配置​

  • 配置结构
    • 字段：模型文件路径、策略文件路径
    • 用途：初始化 Casbin Enforcer 的模型与策略来源
  • 中间件行为
    • 从上下文提取用户名（需前置 JWT 中间件）
    • 从请求提取资源（URL）与动作（HTTP 方法）
    • 调用 Enforce 执行权限判断，返回 403 或放行
  • 令牌结构
    • 包含访问令牌、刷新令牌、作用域、过期时长
  • 生成器
    • 支持分别设置访问令牌与刷新令牌的过期时间
    • 使用 HS256 签名算法
  • 中间件
    • 校验 Authorization 头格式（Bearer）
    • 使用 UserClaims 解析与校验，区分过期、未生效、签名无效等场景
    • 将 Principal 注入上下文供后续使用
  • 角色定义
    • 在 UserClaims 中维护角色数组
  • 权限分配
    • 通过 Casbin 策略文件定义角色与资源/动作的授权关系
  • 资源访问控制
    • 中间件根据用户名、资源、动作执行 Enforce
    • 未授权返回 403
  • 请求头约定
    • 客户端ID、签名(Base64)、随机数、时间戳
  • 防重放与时间窗口
    • 使用 NonceStore 记录随机数及过期时间
    • 校验时间戳是否在允许窗口内
  • 签名验证
    • 使用 SHA256 + RSA-PSS 对拼接数据进行验证
  • 缓存优化
    • 使用 SignatureCache 缓存验证结果，提升性能
  • 配置加载
    • 通过 Load 读取多格式配置文件，自动环境变量注入，校验必要字段
  • 引导层 Authorization
    • 组合 TokenGenerator 与中间件列表，支持命名中间件注册与获取
  • 中间件链路
    • JWT 中间件 → RSA 客户端签名中间件（可选）→ RBAC 中间件 → 业务处理器

  • RSA 签名缓存

    • MemorySignatureCache 提供基于内存的签名验证结果缓存，减少重复计算

  • 随机数清理

    • MemoryNonceStore 与 MemorySignatureCache 内部均启动定时清理 goroutine，避免内存膨胀

  • 建议

    • 生产环境建议替换为分布式缓存（如 Redis）以支持多实例共享
    • 合理设置 TimeWindow 与缓存过期时间，平衡安全性与性能

  • JWT 相关

    • 401 未认证：缺失或格式错误的 Authorization 头
    • 401 令牌无效：格式错误、过期、未生效、签名无效
    • 建议：确认密钥一致、算法匹配、时间同步

  • RBAC 相关

    • 403 权限不足：用户名缺失或策略未授予该资源/动作
    • 建议：核对策略文件与用户名、资源、动作映射

  • RSA 相关

    • 401 缺失/无效参数：请求头缺失或格式错误
    • 401 时间戳过期：超出 TimeWindow
    • 401 重复请求：随机数已存在
    • 401 签名验证失败：数据拼接或签名解码异常
    • 建议：确保请求体完整读取与重置、公钥正确、时间同步

本项目提供了完善的认证与授权基础设施：

  • 通过 JWT 实现用户身份认证与令牌管理
  • 通过 Casbin 实现细粒度的 RBAC 权限控制
  • 通过 RSA-PSS 签名与时间窗口、随机数机制实现客户端签名认证与防重放
  • 配置层支持灵活加载与校验，引导层便于组合中间件与令牌生成器

建议在生产环境中结合分布式缓存与密钥管理服务，进一步强化安全与可运维性。

附录​

  • 安全最佳实践
    • 密码策略：使用强哈希算法与盐值，定期轮换；避免明文存储
    • 会话管理：短生命周期访问令牌 + 长生命周期刷新令牌；启用 HTTPS 与 SameSite Cookie
    • API 保护：统一鉴权中间件、速率限制、输入校验与最小权限原则
    • 密钥管理：密钥分层与轮换、硬件/云密钥管理服务、最小暴露面
    • 日志与审计：记录鉴权事件与异常，保留足够上下文但不泄露敏感信息
