简介​

本文件系统性阐述本仓库中的发布订阅机制，覆盖以下方面：

  • 发布器与订阅器的设计模式与职责边界
  • 主题命名与路由策略、消息分发算法
  • 发布确认与订阅确认流程、回执处理
  • 配置项与性能调优参数
  • 自定义发布器与订阅器的实现示例路径
  • 消息丢失与重复处理策略
  • 最佳实践与常见陷阱

项目结构​

围绕发布订阅，相关模块主要位于 pkg/messaging 下，分为三层：

  • 接口与抽象层：定义发布/订阅、事件读取等接口与泛型约束
  • 适配器层：基于 NATS JetStream 的发布器、订阅器与事件读取器实现
  • 业务集成层：事件发布器（将聚合事件发布到事件总线）、事件订阅器（从事件总线订阅并分发）
  • StreamPublisher：统一的事件发布接口，支持单事件与批量事件发布
  • StreamSubscriber：统一的事件订阅接口，支持 Start/Stop/Close 生命周期
  • StreamReader：事件流读取与重放接口，支持按聚合ID、版本、偏移量重放
  • EventPublisher：将聚合未提交事件发布到事件总线（通用事件模型）
  • EventSubscriber[T]：从事件总线订阅并按主题过滤，反序列化为强类型事件后交由处理器处理
  • JetStreamPublisher：基于 NATS JetStream 的事件发布器，自动确保流存在，主题为 聚合类型.聚合ID.事件类型
  • JetStreamSubscriber[T]：基于 NATS JetStream 的事件订阅器，显式确认、优雅关闭、临时消费者模式
  • JetStreamReader：基于 JetStream 的事件读取器，支持临时消费者、批量拉取、按版本/偏移量重放

发布订阅的整体流程如下：

  • 聚合产生事件 → 事件发布器将事件发布到事件总线（或直接发布到 JetStream）
  • 订阅器注册处理器 → 订阅事件主题 → 消费者从 JetStream 拉取消息 → 反序列化为强类型事件 → 调用领域处理器
  • 事件读取器用于离线重放与聚合重建

发布器设计与职责​

  • EventPublisher
    • 职责：从事件存储读取事件，构造完整主题（服务名.聚合类型.事件类型），将领域事件序列化后发布到事件总线
    • 关键点：主题命名、Base64 负载、逐条发布
  • JetStreamPublisher
    • 职责：确保 JetStream 流存在，按 聚合类型.聚合ID.事件类型 主题发布事件
    • 关键点：流配置 Subjects、Retention 策略、显式确认、批量发布
  • EventSubscriber[T]
    • 职责：初始化时校验参数，按服务名/聚合类型/事件类型编码主题，注册到事件总线；在回调中解码 Base64、反序列化为强类型事件、调用处理器
    • 关键点：主题编码、服务名/聚合类型过滤、处理器注册与并发安全
  • JetStreamSubscriber[T]
    • 职责：创建持久化消费者，Consume 循环处理消息；显式确认；优雅关闭（context 控制、WaitGroup 等待）
    • 关键点：AckPolicy 显式确认、FilterSubject 精确订阅、ReplayPolicy 立即重放、panic recover、状态锁
  • 事件总线（EventPublisher/EventSubscriber）
    • 主题命名：服务名.聚合类型.事件类型
    • 订阅过滤：支持服务名、聚合类型过滤；事件负载 Base64 编码，到达后解码再反序列化
  • JetStream（JetStreamPublisher/JetStreamSubscriber）
    • 主题命名：聚合类型.聚合ID.事件类型
    • 订阅过滤：FilterSubject 聚合类型.*.事件类型，消费者配置 AckPolicy 显式确认、DeliverAllPolicy、ReplayInstantPolicy
    • 事件读取：临时消费者（ephemeral）按聚合ID过滤，支持按版本/偏移量重放
  • 事件总线分发：订阅器注册时生成主题，事件到达后按主题匹配，执行包装处理器进行过滤与反序列化，再调用用户处理器
  • JetStream 分发：消费者按 FilterSubject 订阅，Consume 回调中反序列化 BaseEvent 并转为强类型事件，调用处理器，最后显式确认

  • 发布确认

    • JetStreamPublisher：使用 jetstream.Publish，底层由 NATS JetStream 提供可靠性保证（取决于服务器配置）
    • 事件总线：EventPublisher 发布通用事件，依赖事件总线实现的可靠性

  • 订阅确认

    • JetStreamSubscriber：AckPolicy 显式确认；处理失败时 NAK，触发重试；优雅关闭时等待处理完成
    • 事件总线：EventSubscriber 通过事件总线 Sub/Unsub 管理订阅，处理成功后由上层总线确认

  • JetStream：消息处理成功后显式 Ack；处理失败时 Nak，触发重试；上下文取消时快速 Ack 并返回

  • 事件总线：通过事件总线实现的确认语义（此处由具体事件总线实现决定）

  • JetStreamPublisher 选项

    • WithTopics：设置聚合类型列表，作为流的主题前缀
    • WithMaxAge/WithMaxSize/WithMaxMsgSize：流与消息大小限制
    • Debug：内存存储调试开关

  • JetStreamSubscriber 订阅器

    • 消费者配置：Durable、AckPolicy 显式确认、FilterSubject、DeliverAllPolicy、ReplayInstantPolicy
    • 优雅关闭：Stop(ctx) 支持超时，WaitGroup 等待处理完成

  • 事件读取器

    • 临时消费者：ephemeral，避免持久化消费者带来的控制面压力
    • 批量拉取：FetchNoWait(1000)，按版本排序后应用到聚合

  • 自定义发布器

    • 实现 StreamPublisher 接口：参考 JetStreamPublisher 的 Publish/PublishEvents 与 ensureStream 流创建逻辑
    • 示例路径：JetStreamPublisher, JetStreamPublisher.ensureStream, JetStreamPublisher.Publish

  • 自定义订阅器

    • 实现 StreamSubscriber 接口：参考 JetStreamSubscriber 的 Start/Stop/Close 与 runConsumer/handleMessage
    • 示例路径：JetStreamSubscriber.Start, JetStreamSubscriber.Stop, JetStreamSubscriber.handleMessage

  • 事件总线桥接

    • EventPublisher：参考 EventPublisher, EventPublisher.publishEvent
    • EventSubscriber[T]：参考 EventSubscriber[T], EventSubscriber[T].AddServiceHandler

  • 丢失处理

    • JetStream：使用显式确认（AckPolicy 显式确认），处理失败时 Nak 触发重试；优雅关闭时等待处理完成
    • 事件总线：依赖事件总线实现的可靠性语义

  • 重复处理

    • 订阅器应具备幂等性设计；事件读取器支持按版本重放，便于幂等应用
    • 事件总线桥接：EventPublisher 发布前可结合事件存储顺序性，减少重复概率

  • 最佳实践

    • 一个服务仅维护一个 JetStreamPublisher 实例，流名为服务名，主题为聚合类型.>
    • 订阅器使用显式确认，处理逻辑尽量短小、幂等
    • 使用临时消费者进行事件读取，避免持久化消费者带来的控制面压力
    • 事件总线桥接时统一主题命名规范，便于跨服务解耦

  • 常见陷阱

    • 多个订阅器并行运行导致事件处理无序与不一致
    • 忘记 Ack 导致消息堆积与重复重试
    • 主题命名不一致导致事件无法正确路由

发布订阅组件之间的依赖关系如下：

  • 消费者批处理：事件读取器使用 FetchNoWait(1000) 批量拉取，降低控制面开销

  • 临时消费者：避免持久化消费者带来的控制面压力

  • 确认策略：显式确认避免重复处理，合理设置重试策略

  • 主题设计：聚合类型.> 作为主题前缀，便于扩展与隔离

  • 发布失败

    • 检查 JetStream 流是否存在与配置是否正确
    • 查看日志输出，定位序列化/发布错误

  • 订阅无响应

    • 确认主题命名与 FilterSubject 是否匹配
    • 检查消费者是否创建成功、是否处于运行状态

  • 处理失败重试

    • 检查 Nak 与重试策略；确保处理器幂等

  • 优雅关闭超时

    • 检查 WaitGroup 是否阻塞；确认上下文取消传播

本发布订阅体系通过统一接口抽象与 JetStream 适配器，提供了高可靠、可扩展的消息发布与订阅能力。结合事件总线桥接与事件读取器，能够满足事件驱动架构下的事件发布、订阅、重放与聚合重建需求。遵循主题命名规范、显式确认与幂等处理，是保障系统稳定性的关键。
