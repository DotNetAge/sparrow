简介​

本文件系统性阐述并发调度器的设计与实现，覆盖并发任务管理、任务池控制、资源限制、负载均衡、任务调度流程、执行监控、错误处理与重试机制，并给出性能优化建议、资源使用监控与故障恢复策略。重点解释 SetMaxConcurrentTasks 的工作机制与配置参数，提供实际使用示例与最佳实践。

项目结构​

并发调度器位于 pkg/tasks 目录，围绕统一的 TaskScheduler 接口实现多种执行模式：

  • 并发执行模式：concurrent_scheduler.go
  • 顺序执行模式：sequential_scheduler.go
  • 混合执行模式：hybrid_scheduler.go
  • 任务构建与包装：builder.go、wrapper.go
  • 核心类型与常量：core.go
  • 测试与示例：tasks_test.go
  • 应用启动与集成：app.go
  • TaskScheduler 接口：统一的调度器能力抽象，包含调度、取消、状态查询、列表、启动、停止、关闭等。
  • Task 接口：任务契约，包含 ID、类型、调度时间、处理器、完成/取消回调、周期性与超时等。
  • TaskInfo：任务元数据，含状态、调度时间、创建/更新时间、重试计数与最大重试、下次重试时间、TTL 等。
  • TaskBuilder：任务构建器，提供链式配置（ID、类型、处理器、回调、调度、超时、重试、优先级）。
  • SchedulerWrapper：调度器包装器，作为对外唯一入口，负责任务提交、配置与模式选择、并发数设置、状态查询与列表。
  • 并发调度器 concurrentScheduler：支持多任务并发执行，内置活跃任务计数与条件变量，支持任务过期清理、定时/重试任务处理、超时与重试。
  • 顺序调度器 sequentialScheduler：串行执行，保证任务顺序，内置队列与通知通道，支持定时/周期任务与指数退避重试。
  • 混合调度器 HybridScheduler：根据任务类型或调度特性选择并发或顺序执行，统一对外接口。

并发调度器采用“接口抽象 + 多实现 + 包装器入口”的分层设计：

  • 接口层：TaskScheduler、Task、TaskInfo 定义统一契约。
  • 实现层：concurrentScheduler、sequentialScheduler、HybridScheduler 分别实现不同执行策略。
  • 工具层：TaskBuilder、SchedulerWrapper 提供易用的构建与调用入口。
  • 生命周期：Start/Stop/Close 提供一致的启动与优雅关闭语义；并发调度器通过 goroutine + 条件变量实现并发槽位控制与任务监控。

并发调度器 concurrentScheduler​

  • 并发控制：通过活跃任务计数 activeTasks 与条件变量 activeTasksCond 控制并发槽位；执行前检查并等待，完成后减少计数并通知。
  • 任务监控：runTaskMonitor 定时扫描定时/重试任务并触发执行；cleanupExpiredTasks 清理过期已完成任务。
  • 任务池与限制：maxConcurrent 控制最大并发；maxTaskCount 限制任务总数，removeOldestCompletedTask 选择最老已完成任务淘汰。
  • 任务执行：executeTask 中设置运行中状态、支持任务超时、错误分类（超时/取消 vs 失败）、重试计数与退避、死信处理、完成回调。
  • 状态管理：TaskInfo 状态机覆盖 waiting/running/completed/cancelled/failed/retrying/dead_letter。
  • 功能：动态设置并发调度器的最大并发任务数，仅对并发调度器与混合调度器内的并发部分生效。
  • 参数校验：要求大于 0，否则返回错误。
  • 并发安全：使用互斥锁保护 maxConcurrent 字段。
  • 生效路径：
    • 直接并发调度器：通过包装器的 SetMaxConcurrentTasks 调用并发调度器的 SetMaxConcurrentTasks。
    • 混合调度器：通过包装器设置并发部分的 maxConcurrent。
  • 执行等待：并发调度器在执行前轮询检查 activeTasks 与 maxConcurrent，必要时短暂休眠等待槽位释放。
  • 调度入口：Schedule(task) 校验任务合法性，设置 TaskInfo 状态与时间戳，存储任务；若为即时或已到期则异步执行。
  • 定时/重试：runTaskMonitor 周期扫描，匹配 waiting/timed 与 retrying/nextRetryAt 到期任务，触发 executeTask。
  • 执行监控：executeTask 中：
    • 设置运行中状态
    • 支持任务超时（context.WithTimeout）
    • 成功：标记 completed；周期性任务按间隔重新调度
    • 失败：区分超时/取消与业务错误；按 MaxRetries 与 RetryCount 执行线性退避重试；超过阈值进入 dead_letter
    • 触发 OnComplete 回调
  • 过期清理：cleanupExpiredTasks 基于 TTL 清理已完成/取消/死信任务，避免内存膨胀。

  • 错误分类：

    • 超时/取消：context.Canceled 或 context.DeadlineExceeded，标记 cancelled，不重试。
    • 业务错误：若 MaxRetries > 0 且 RetryCount < MaxRetries，进入 retrying，线性退避（100ms+100ms×次），上限 500ms；否则进入 dead_letter。

  • 重试退避：线性增长，最大 500ms；并发调度器当前未实现指数退避，顺序调度器实现了指数退避。

  • 死信处理：超过最大重试次数或不可重试时，标记 dead_letter 并保留 LastError。

  • 顺序执行：isRunning + pendingTasks 队列 + taskDoneChan 通知，严格串行。

  • 定时/周期：runTaskMonitor 扫描到期任务并加入队列；executeTask 中支持指数退避重试与周期性任务重新调度。

  • 并发数：SetMaxConcurrentTasks 对顺序调度器无效（始终为1）。

  • 类型映射：WithConcurrent/WithSequential 将任务类型映射到执行模式。

  • 选择策略：按类型映射优先；定时/周期任务默认顺序；其他默认并发。

  • 统一接口：Start/Stop/Close 分别委托给并发与顺序调度器；Schedule/Cancel/GetTaskStatus/ListTasks 合并结果。

  • 任务提交：RunTask/RunTaskWithID/RunTaskAt/RunTaskRecurring/RunTypedTask 等便捷方法。

  • 并发数设置：SetMaxConcurrentTasks 通过类型断言调用底层并发调度器。

  • 模式配置：WithMaxConcurrent、WithConcurrent、WithSequential；默认使用混合调度器。

  • 默认重试：buildTaskWithRetry 注入默认重试次数。

  • 并发调度器依赖 Task 接口与 TaskInfoProvider 接口以读写任务元数据。

  • 混合调度器组合并发与顺序调度器，实现按类型/调度特性的模式选择。

  • 包装器组合调度器与构建器，提供统一入口。

  • 测试覆盖并发数限制、重试、超时、内存管理等关键行为。

  • 并发槽位控制：并发调度器采用轮询 + 短暂休眠的方式等待槽位释放，避免忙等；建议根据 CPU/IO 特性调整 maxConcurrent。
  • 任务过期清理：定期清理已完成/取消/死信任务，降低内存占用；可通过 taskTTL 与 maxTaskCount 控制。
  • 重试退避：线性退避（并发）或指数退避（顺序），避免雪崩效应；建议结合业务场景选择合适策略。
  • 资源监控：建议在应用层记录活跃任务数、执行耗时、重试次数、死信比例等指标，结合日志与告警。
  • 负载均衡：混合调度器按任务类型分流，可将高吞吐任务走并发，关键顺序任务走顺序，提升整体吞吐与一致性。

故障排查指南​

  • 任务长时间处于 waiting：检查 Schedule 时间是否正确、runTaskMonitor 是否正常运行。
  • 任务频繁重试：确认 MaxRetries 与退避策略；查看 LastError 与 NextRetryAt。
  • 超时/取消：检查任务 Handler 是否正确处理 context；合理设置 GetTimeout。
  • 死信堆积：核对 dead_letter 任务数量与原因；考虑放宽重试或引入死信队列。
  • 并发不足：调用 SetMaxConcurrentTasks 提升并发；观察活跃任务峰值与系统资源。
  • 优雅关闭：Stop/Close 会等待活跃任务完成；确保任务 Handler 支持 context 取消。

并发调度器通过清晰的接口抽象与多执行模式实现，提供了灵活的任务编排能力。并发调度器在高吞吐场景表现优异，配合合理的并发数、重试与过期清理策略，可在保证稳定性的同时最大化吞吐。混合调度器进一步提升了任务类型的差异化处理能力。建议在生产环境中结合监控与告警，持续优化并发参数与重试策略。

实际使用示例与最佳实践​

  • 快速开始：使用 SchedulerWrapper 默认构造并发调度器，提交任务、查询状态、取消任务。

  • 设置并发数：通过包装器的 WithMaxConcurrent 或直接调用 SetMaxConcurrentTasks 设置并发上限。

  • 任务类型与模式：使用 WithConcurrent/WithSequential 为任务类型指定执行模式；定时/周期任务默认顺序。

  • 重试与超时：通过 TaskBuilder.WithRetry 与 GetTimeout 配置；注意区分超时/取消与业务错误。

  • 监控与告警：记录活跃任务数、执行耗时、重试次数、死信比例；结合日志与指标系统。

  • 并发调度器

    • Start(ctx)、Stop()、Close(ctx)

    • RunTask/RunTaskWithID/RunTaskAt/RunTaskRecurring/RunTypedTask
    • SetMaxConcurrentTasks(max)、SetTaskTypeMode(taskType, mode)
    • GetTaskStatus(taskID)、ListTasks()、CancelTask(taskID)
