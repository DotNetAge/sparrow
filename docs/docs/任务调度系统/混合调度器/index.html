项目结构​

混合调度器位于任务模块(pkg/tasks)，围绕统一接口TaskScheduler展开，包含三种核心实现：

  • 混合调度器(HybridScheduler)：组合并发与顺序调度器，按任务类型与配置进行路由
  • 并发调度器(ConcurrentScheduler)：支持多任务并发执行，具备内存管理、超时与重试
  • 顺序调度器(SequentialScheduler)：串行执行，适合定时与周期任务
  • 任务接口与状态
    • TaskScheduler：统一的调度器接口，定义调度、取消、状态查询、列表、并发上限设置、启动/停止/关闭等能力
    • Task/TaskInfo/TaskInfoProvider：任务抽象、任务信息与信息提供接口
    • TaskStatus/ExecutionMode：任务状态与执行模式枚举
  • 调度器实现
    • HybridScheduler：组合并发与顺序调度器，按任务类型与配置进行路由
    • ConcurrentScheduler：并发执行，支持并发槽位控制、定时/重试任务处理、过期清理
    • SequentialScheduler：顺序执行，支持定时任务触发、周期任务重排、过期清理
  • 任务构建与包装
    • TaskBuilder：任务构建器，支持ID、类型、处理器、回调、调度时间、超时、重试、优先级等
    • SchedulerWrapper：调度器包装器，提供便捷的提交接口、默认重试、并发上限设置、任务类型模式映射
  • 应用集成
    • App：应用实例，持有SchedulerWrapper
    • Tasks()：启动器选项，将SchedulerWrapper注入App

混合调度器通过统一接口TaskScheduler对外提供能力，内部组合并发与顺序两种执行模式的调度器。其核心特性包括：

  • 任务路由：根据任务类型与配置决定使用并发还是顺序执行；定时与周期任务默认顺序执行
  • 并发控制：并发调度器维护活跃任务计数与条件变量，限制最大并发数
  • 顺序控制：顺序调度器维护待执行队列、执行器与监视器，保证串行执行
  • 状态管理：统一的任务信息(TaskInfo)与状态(TaskStatus)，支持重试、死信、过期清理
  • 包装器集成：SchedulerWrapper提供便捷API与默认配置，App通过Tasks()选项集成

混合调度器(HybridScheduler)​

  • 设计要点
    • 组合模式：持有并发与顺序两个调度器实例
    • 执行模式映射：通过executionModes将任务类型映射到执行模式，支持并发/顺序
    • 路由策略：优先检查任务类型映射；若为定时或周期任务，强制顺序执行；否则并发执行
    • 统一生命周期：Start/Stop/Close分别委托给两个子调度器
  • 关键方法
    • WithConcurrent/WithSequential：配置任务类型到执行模式的映射
    • Schedule：根据规则选择调度器并执行
    • Cancel/GetTaskStatus/ListTasks：在两个调度器中查找/操作
    • SetMaxConcurrentTasks：仅对并发调度器生效
  • 线程安全：执行模式映射使用读写锁保护
  • 设计要点
    • 并发执行：多个任务可同时运行，通过活跃任务计数与条件变量控制并发槽位
    • 任务监控：定时器周期扫描定时任务与重试任务，触发执行
    • 内存管理：限制最大任务数量，按更新时间清理最老的已完成/取消/死信任务
    • 超时与重试：支持任务超时，失败时按最大重试次数与线性退避策略重试，超过阈值进入死信
  • 关键方法
    • Start：启动任务监视器协程
    • Schedule：存储任务，必要时立即执行
    • Cancel：更新状态并触发取消回调
    • Stop/Close：等待活跃任务完成或优雅关闭
    • SetMaxConcurrentTasks：设置最大并发数
    • processScheduledTasks/cleanupExpiredTasks：定时处理与清理
    • executeTask：执行单个任务，处理超时、重试与完成回调
  • 设计要点
    • 串行执行：同一时刻仅有一个任务在执行，保证顺序一致性
    • 待执行队列：pendingTasks维护待执行任务ID，taskQueue提供缓冲
    • 任务执行器：按序取出任务执行，支持超时与重试
    • 监视器：定时检查定时任务是否到达执行时间，清理过期任务
    • 周期任务：完成后重新计算下次执行时间并回填队列
  • 关键方法
    • runTaskExecutor：循环查找等待中的定时/到期任务并执行
    • runTaskMonitor：定时清理过期任务与触发定时任务
    • executeTask：执行单个任务，处理超时、重试与完成回调
    • cleanupExpiredTasks/cleanupOldTasks：过期与旧任务清理
  • 设计要点
    • SchedulerWrapperOptions：默认重试次数、调度器实例、任务类型模式映射、最大并发数
    • NewSchedulerWrapper：根据选项创建默认混合调度器，应用并发上限与任务类型映射
    • RunTask系列：封装TaskBuilder，支持立即、定时、周期、延迟执行
    • SetTaskTypeMode/SetMaxConcurrentTasks：动态调整混合调度器的模式与并发上限
  • 与应用集成
    • Tasks()选项：将SchedulerWrapper注入App，实现任务系统的启动与清理

  • 设计要点

    • 支持ID、类型、处理器、完成/取消回调、调度时间、超时、重试次数、优先级
    • Build：校验参数合法性，生成builtTask并填充TaskInfo
    • builtTask：实现Task与TaskInfoProvider接口，提供任务信息访问

  • 与调度器配合

    • SchedulerWrapper通过TaskBuilder构建任务，统一注入默认重试次数

  • 接口与实现

    • TaskScheduler为统一接口，HybridScheduler/ConcurrentScheduler/SequentialScheduler均实现该接口
    • Task与TaskInfoProvider为任务与任务信息的契约

  • 包装器与调度器

    • SchedulerWrapper持有TaskScheduler实例，提供高层API
    • HybridScheduler组合ConcurrentScheduler与SequentialScheduler

  • 应用集成

    • App持有SchedulerWrapper，Tasks()选项将其注入应用生命周期
  • 并发上限控制
    • ConcurrentScheduler通过活跃任务计数与条件变量控制并发，避免资源争用
    • SchedulerWrapper.SetMaxConcurrentTasks设置最大并发数，HybridScheduler仅影响并发部分
  • 内存与过期清理
    • 任务数量达到上限时，移除最老的已完成/取消/死信任务
    • 过期任务按TTL清理，避免内存膨胀
  • 重试与退避
    • 并发调度器采用线性退避，顺序调度器采用指数退避，避免雪崩效应
  • 监控与可观测性
    • 通过ListTasks与GetTaskStatus获取任务状态，便于监控与告警
  • 调优建议
    • 根据业务峰值合理设置最大并发数
    • 对高吞吐任务使用并发模式，对强一致需求使用顺序模式
    • 合理配置重试次数与退避策略，避免频繁重试导致抖动
    • 定期清理过期任务，维持系统健康

故障排查指南​

  • 任务状态异常
    • 使用GetTaskStatus确认任务状态，结合ListTasks定位问题
  • 并发阻塞
    • 检查活跃任务数与最大并发设置，确认是否存在长时间运行任务
  • 任务未执行
    • 检查任务类型映射与调度时间，确认是否被路由到顺序模式
  • 重试失败
    • 查看重试次数与退避策略，确认是否达到最大重试
  • 资源泄漏
    • 确认Stop/Close调用，确保调度器优雅关闭

混合调度器通过统一接口与组合模式，实现了并发与顺序调度的无缝衔接。其智能路由、并发控制、顺序保障与完善的生命周期管理，使其适用于多样化的任务调度场景。配合SchedulerWrapper与应用启动器，可在实际业务中快速落地并进行灵活调优。

配置方法与使用场景​

  • 配置方法

    • 通过SchedulerWrapperOption设置默认重试次数、最大并发数、任务类型模式映射
    • 通过Tasks()选项将SchedulerWrapper注入App，实现任务系统启动与清理

  • 使用场景

    • 高吞吐异步任务：并发模式
    • 强一致定时任务：顺序模式
    • 混合场景：混合调度器按类型路由

  • 并发/顺序混合执行

    • 通过RunTypedTaskWithID提交不同类型的并发/顺序任务，验证执行模式

  • 定时/周期任务

    • 通过RunTaskAt/RunTaskRecurring提交定时/周期任务，顺序调度器负责严格顺序

  • 重试与死信

    • 通过默认重试配置与状态检查，观察重试与死信行为
