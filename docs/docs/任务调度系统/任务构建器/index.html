简介​

任务构建器是 Sparrow 任务管理系统的核心组件，采用 Fluent API 设计模式，提供了简洁直观的任务配置和创建接口。该系统支持多种执行模式（即时、定时、周期性），具备完善的错误处理、重试机制和超时控制功能。

任务构建器系统由以下核心组件构成：

1. 任务构建器 (TaskBuilder)​

采用 Fluent API 设计模式，提供链式调用接口：

  • 任务ID设置：WithID()
  • 类型定义：WithType()
  • 处理函数：WithHandler()
  • 回调函数：WithOnComplete()、WithOnCancel()
  • 调度配置：WithSchedule()、WithRecurring()
  • 执行参数：WithTimeout()、WithRetry()、WithPriority()

2. 核心接口和类型​

  • Task 接口：定义任务的标准行为
  • TaskScheduler 接口：定义调度器的标准行为
  • TaskInfo 结构：封装任务的元数据
  • TaskSchedule 结构：定义任务的调度配置

3. 调度器实现​

  • 并发调度器：支持多任务并行执行
  • 顺序调度器：严格按序执行任务
  • 混合调度器：根据任务类型智能选择执行模式

故障排除指南​

常见问题及解决方案​

任务构建失败​

问题：任务处理器不能为空 原因：未设置任务处理函数 解决：确保调用 WithHandler() 方法

时间配置错误​

问题：定时任务的执行时间必须大于当前时间 原因：设置过去的时间点 解决：使用未来的时间点或使用 WithRecurring()

并发数配置问题​

问题：最大并发任务数必须大于0 原因：设置非正数 解决：确保并发数大于0

任务取消异常​

问题：任务不存在 原因：任务ID错误或任务已完成 解决：检查任务ID和任务状态

  1. 启用详细日志记录
  2. 使用 ListTasks() 检查任务状态
  3. 监控内存使用情况
  4. 验证重试配置是否合理

结论​

任务构建器系统通过 Fluent API 设计模式，提供了简洁直观的任务配置接口。系统支持多种执行模式，具备完善的错误处理和重试机制。通过选项模式配置，用户可以灵活地定制调度器的行为。系统的模块化设计使得扩展和维护变得简单。
