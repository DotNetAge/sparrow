引言​

本文件聚焦“业务规则”的建模与实现，面向业务分析师与开发者，系统阐述如何在领域模型中定义、封装与执行复杂业务规则，覆盖任务状态转换、会话生命周期、聚合一致性与事件溯源等主题。文档同时给出规则验证时机、错误处理策略、规则链设计、边界情况与演进管理的实践建议，并通过图示化的方式帮助读者快速把握关键流程。

项目结构​

本项目采用整洁架构分层，业务规则主要集中在“实体层”和“用例层”，并通过接口适配器层对外暴露能力。与业务规则直接相关的关键模块如下：

  • 实体层：任务、会话、事务与聚合根基类、领域事件接口
  • 用例层：会话服务、Saga事务服务
  • 错误层：统一错误类型定义
  • 文档与路由：聚合根模板使用说明、HTTP路由注册
  • 任务实体：定义任务状态枚举与状态转换方法，内置重试、过期、执行时长等规则。
  • 会话实体：定义会话生命周期规则（过期、续期、数据读写）。
  • 聚合根接口与基类：提供事件溯源、版本控制、未提交事件管理与一致性保障。
  • 领域事件接口：统一事件元数据，支撑事件溯源与投影。
  • Saga事务实体与服务：定义事务步骤与补偿机制，实现跨边界的最终一致性。
  • 用例服务：封装业务规则的执行时机与错误处理，协调实体与仓储。
  • 错误类型：统一错误语义，便于规则失败时的诊断与恢复。

业务规则在整洁架构中的分布与交互如下：

  • 实体层承载核心业务规则与不变量。
  • 用例层编排规则执行，处理错误与边界条件。
  • 接口适配器层接收外部请求，调用用例层。
  • 错误层提供统一错误语义，便于上层处理。

任务状态转换规则​

任务实体定义了明确的状态枚举与转换方法，确保状态变迁满足业务约束：

  • 初始状态：待处理
  • 可达状态：运行中、已完成、失败、取消
  • 关键规则：
    • 失败后可按最大重试次数进行重试
    • 过期规则：创建时间超过24小时即视为过期
    • 执行时长：仅当起止时间均存在时计算

会话实体与会话服务共同定义了会话的生命周期规则：

  • 创建：指定ID与TTL，初始化数据容器与过期时间
  • 访问：获取会话时若已过期则删除并返回未找到
  • 更新：保存会话时自动续期
  • 数据：提供键值存取与删除
  • 批量清理：遍历并删除过期会话

聚合根接口与基类提供了事件溯源、版本控制与一致性保障：

  • 版本控制：乐观锁，防止并发覆盖
  • 事件管理：未提交事件列表，提交后清空
  • 事件校验：事件ID/类型/聚合ID合法性检查，避免重复事件ID
  • 业务验证：预留Validate接口，可在聚合根中扩展复杂规则

事务实体与服务定义了跨边界的事务执行与补偿机制：

  • 事务状态：待执行、执行中、已完成、失败、已补偿
  • 执行流程：按步骤顺序执行，失败时反向补偿
  • 补偿策略：从失败步骤向前逐个执行补偿处理器

  • 验证时机

    • 实体内聚：在实体方法内部即时验证（如任务状态转换、会话过期检查）
    • 用例编排：在用例服务中进行跨实体的业务规则校验（如会话访问前的过期检查）
    • 事件溯源：在聚合根ApplyEvent中确保事件应用后的状态一致性

  • 封装策略

    • 将规则封装为实体方法（如实体的IsExpired、CanRetry）
    • 将跨实体规则封装为用例服务方法（如会话服务的GetSession）
    • 将复杂规则拆分为独立的校验器或策略对象（推荐在用例层实现）

  • 规则链

    • 会话访问链：查找 -> 过期检查 -> 删除过期 -> 返回结果
    • 任务执行链：Start -> 执行业务逻辑 -> Complete/Fail -> 可能重试
    • Saga链：顺序执行步骤 -> 失败补偿（逆序）

  • 条件判断

    • 时间比较：会话过期、任务过期
    • 计数比较：重试次数与最大重试
    • 状态机：任务状态转换的合法路径

  • 会话

    • 未找到：过期会话删除后返回未找到错误
    • 无效会话：保存空会话返回无效会话错误

  • 仓储

    • 未找到实体、非法实体、操作失败等统一包装为仓储错误

  • 事件存储

    • 并发冲突、聚合不存在、自定义错误类型

  • 快照

    • 未找到快照错误

  • 测试方法

    • 单元测试：针对实体方法（如IsExpired、CanRetry）与用例服务方法（如GetSession、ExecuteTransaction）
    • 集成测试：通过HTTP路由与处理器调用端到端验证业务规则
    • 并发测试：验证事件存储的乐观锁与版本控制

  • 规则演进

    • 通过聚合根的Validate接口扩展复杂业务验证
    • 通过事件溯源的LoadFromEvents与ApplyEvent实现规则变更后的状态重放
    • 通过快照优化大规模历史事件的重放性能

  • 实体依赖基础实体以获得标准字段与时间戳

  • 聚合根接口与基类提供事件溯源与版本控制能力

  • 用例服务依赖实体与仓储，向外暴露业务规则

  • 错误类型为各层提供一致的错误语义

  • 事件溯源与快照：对历史事件过多的聚合，建议定期生成快照以减少重放成本
  • 并发控制：通过聚合根版本号实现乐观锁，避免写冲突导致的重试风暴
  • 会话续期：保存会话时自动续期，降低频繁访问导致的过期风险
  • Saga执行：顺序步骤的失败补偿应尽量幂等，减少补偿失败带来的额外开销

故障排查指南​

  • 会话相关
    • 404未找到：确认会话是否过期被清理；检查ID是否正确
    • 无效会话：检查传入参数是否为空
  • 事件存储
    • 并发冲突：检查期望版本与当前版本，必要时重试或回滚
    • 聚合不存在：确认聚合ID是否存在
  • 仓储
    • 未找到实体：确认查询条件与索引
    • 操作失败：查看底层存储异常
  • 快照
    • 未找到快照：确认快照是否已生成或被清理

本项目通过清晰的分层与强约束的实体/聚合根设计，将复杂的业务规则内聚在领域模型中，并通过用例层进行编排与错误治理。任务状态转换、会话生命周期与Saga事务补偿等规则均有明确的实现与验证路径。建议在实际落地中：

  • 将规则封装为实体方法与用例服务方法，保持高内聚低耦合
  • 通过事件溯源与快照保障规则演进与状态一致性
  • 建立完善的测试体系，覆盖正常流程、边界与异常场景
