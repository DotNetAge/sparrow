简介​

本指南面向首次接触 Sparrow 框架的新用户，帮助你在最短时间内完成环境准备、项目克隆、依赖安装与配置，并运行一个最小可用的“Hello World”微服务示例。文档同时提供常见问题排查建议与多平台安装注意事项，确保你能顺利上手。

项目结构​

Sparrow 采用模块化的分层架构，核心目录与职责概览如下：

  • pkg/bootstrap：应用启动、容器、HTTP 服务器与生命周期管理
  • pkg/config：配置加载、默认值与校验
  • pkg/entity/usecase：领域模型与用例层
  • pkg/adapter/http：HTTP 路由与中间件
  • pkg/logger、pkg/messaging、pkg/tasks：日志、消息总线与任务调度等基础设施
  • 根目录：go.mod、makefile 等构建与运行脚本

  • IoC 容器 Container：支持匿名与具名依赖注册与解析，简化服务装配

  • 配置系统：Viper 加载多格式配置文件、环境变量覆盖、默认值与校验

  • 领域实体与用例：Session、Task 等实体及对应的用例服务，体现整洁架构核心层

故障排除指南​

  • 端口占用
    • 现象：启动失败提示端口不可用
    • 处理：修改配置中的 server.port 或释放占用端口
    • 参考：配置默认端口与校验逻辑
  • 配置文件未生效
    • 现象：环境变量覆盖无效或默认值未生效
    • 处理：确认配置文件格式与路径，确保 .env 已正确加载，键名使用下划线风格
    • 参考：配置加载与环境变量替换
  • 依赖注入解析失败
    • 现象：ResolveByName 报错“未找到提供者”
    • 处理：确认已 Register/RegisterNamed 对应构造函数，名称匹配
    • 参考：容器注册与解析逻辑
  • 优雅关闭异常
    • 现象：SIGINT/SIGTERM 后资源未释放
    • 处理：确保通过 NeedCleanup 注册子进程，检查 Close 实现
    • 参考：App 清理流程

环境搭建步骤​

  • 安装 Go 1.25

    • 参考：模块文件中的 Go 版本要求

  • 安装必要工具

    • Git（克隆仓库）
    • Make（执行构建与运行脚本）
    • golangci-lint（可选，代码检查）

  • 安装可选中间件

    • NATS、Redis、PostgreSQL（根据配置需求）

  • 克隆仓库

    • 使用 Git 克隆到本地工作区

  • 安装依赖

    • 执行 make deps（go mod tidy + go mod download）

  • 验证安装

    • 执行 make test 运行单元测试
    • 执行 make run 或 make dev 启动服务

  • 配置文件位置与格式

    • 支持 yaml/yml/json/toml/properties 等格式
    • 搜索路径：当前目录、./configs、/etc/<app>/、$HOME/.<app>/

  • 默认值与环境变量

    • 默认值集中在配置文件中定义
    • 环境变量通过 AutomaticEnv 自动生效，键名替换为下划线风格

  • 关键配置项

    • 应用：app.name、app.version、app.secret、app.key_path
    • 服务器：server.host、server.port
    • 日志：log.level、log.format、log.output、log.filename、log.mode
    • NATS：nats.url、nats.stream_name、nats.max_age
    • Redis：redis.host、redis.port、redis.password、redis.db、redis.es_db
    • Badger：badger.data_dir、badger.es_dir、badger.value_threshold、badger.num_compactors、badger.in_memory
    • Casbin：casbin.model_path、casbin.policy_path
    • SQL：sql.driver、sql.host、sql.port、sql.user、sql.password、sql.dbname、sql.es_dbname
    • RabbitMQ：rabbitmq.host、rabbitmq.port、rabbitmq.username、rabbitmq.password、rabbitmq.vhost、rabbitmq.exchange

以下步骤帮助你创建一个仅输出“Hello World”的最小微服务：

  • 新建入口文件（例如 cmd/main.go）
    • 初始化 App：调用 NewApp 并传入必要选项（如 ServerPort）
    • 注册路由：在 App.Engine 上挂载 GET /hello 返回“Hello World”
    • 启动服务：调用 App.Start()
  • 运行
    • 执行 make run 或 make dev
    • 访问 http://localhost:8080/hello 验证响应

提示

  • 若需自定义端口，请在配置中设置 server.port 或通过环境变量覆盖

  • 如需接入日志、中间件或认证，可参考 App 的 Use 选项与中间件集成方式

  • macOS/Linux

    • 使用包管理器安装 Go 1.25、Git、Make
    • 可选：安装 golangci-lint 进行静态检查

  • Windows

    • 使用官方安装包安装 Go 1.25
    • 使用 Git Bash 或 WSL 获取类 Unix 的 make 体验
    • 注意：配置文件路径与环境变量键名替换规则同样适用

  • Docker

    • 可使用 make docker-build 与 make docker-run 构建并运行容器
    • 注意：.env 中的 APP_NAME 与 APP_VERSION 会影响镜像标签
