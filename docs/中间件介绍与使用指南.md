## JWT 身份认证中间件

### 功能介绍

`JWTAuthMiddleware` 是一个基于 JWT (JSON Web Token) 的身份认证中间件，用于验证请求的合法性并提取用户信息。它能够：

- 从请求头中提取和验证 JWT 令牌
- 支持多种令牌验证错误类型的识别和返回
- 验证通过后将用户信息（用户名、用户ID、角色）存入 Gin 上下文

### 适用场景

适用于需要进行用户身份认证的 API 接口，特别是微服务架构中的服务间通信和需要会话状态的 Web 应用。

### 使用方法

```go
import (
	"github.com/DotNetAge/sparrow/pkg/adapter/http/middlewares"
	"github.com/gin-gonic/gin"
)

func main() {
	// 创建 Gin 引擎
	router := gin.Default()
	
	// 配置 JWT 密钥
	jwtSecret := []byte("your-secret-key-here")
	
	// 应用 JWT 中间件到需要认证的路由组
	authGroup := router.Group("/api")
	authGroup.Use(middlewares.JWTAuthMiddleware(jwtSecret))
	{
		// 受保护的路由
		authGroup.GET("/user/profile", getUserProfile)
		authGroup.POST("/user/update", updateUserProfile)
	}
	
	// 启动服务器
	router.Run(":8080")
}

// 在处理函数中获取用户信息
getUserProfile(c *gin.Context) {
	// 从上下文中获取用户信息
	username, _ := c.Get("username")
	userID, _ := c.Get("userID")
	roles, _ := c.Get("roles")
	
	c.JSON(200, gin.H{
		"username": username,
		"userID": userID,
		"roles": roles,
	})
}
```

### 错误处理

中间件会在不同的错误情况下返回相应的状态码和错误消息：

- 缺少 Authorization 头：返回 401 状态码和 "请求头中缺少Authorization字段" 错误
- Authorization 格式错误：返回 401 状态码和 "Authorization字段格式错误，应为 Bearer <token>" 错误
- 令牌格式错误：返回 401 状态码和 "令牌格式错误 (Malformed)" 错误
- 令牌过期或尚未生效：返回 401 状态码和 "令牌已过期或尚未生效 (Expired/Not Valid Yet)" 错误
- 签名无效：返回 401 状态码和 "令牌签名无效 (Invalid Signature)" 错误
- 其他验证错误：返回 401 状态码和 "无效的令牌 (Invalid Token)" 错误

## RBAC 权限控制中间件

### 功能介绍

`RBACMiddleware` 是一个基于 Casbin 实现的 RBAC (基于角色的访问控制) 中间件，用于控制用户对资源的访问权限。它能够：

- 从上下文获取用户名（依赖 JWT 中间件）
- 获取请求的资源（URI）和动作（HTTP 方法）
- 使用 Casbin 执行权限检查
- 根据权限检查结果允许或拒绝请求

### 适用场景

适用于需要细粒度访问控制的应用，特别是有多级用户角色和复杂权限规则的系统。

### 使用方法

```go
import (
	"github.com/DotNetAge/sparrow/pkg/adapter/http/middlewares"
	"github.com/casbin/casbin/v2"
	"github.com/gin-gonic/gin"
)

func main() {
	// 创建 Gin 引擎
	router := gin.Default()
	
	// 配置 JWT 密钥
	jwtSecret := []byte("your-secret-key-here")
	
	// 初始化 Casbin 授权器
	// 使用文件加载策略
	enforcer, err := casbin.NewEnforcer("path/to/model.conf", "path/to/policy.csv")
	if err != nil {
		// 处理错误
	}
	
	// 应用中间件，注意顺序：先认证后授权
	authGroup := router.Group("/api")
	authGroup.Use(middlewares.JWTAuthMiddleware(jwtSecret))
	authGroup.Use(middlewares.RBACMiddleware(enforcer))
	{
		// 受保护的路由
		authGroup.GET("/admin/users", listUsers)       // 需要管理员权限
		authGroup.POST("/user/update", updateUser)      // 需要用户自身权限
	}
	
	// 启动服务器
	router.Run(":8080")
}
```

### 错误处理

中间件会在不同的错误情况下返回相应的状态码和错误消息：

- 未认证（上下文中没有用户名）：返回 401 状态码和 "未认证" 错误
- 权限检查失败：返回 500 状态码和 "权限检查失败" 错误
- 权限不足：返回 403 状态码和 "权限不足" 错误

## RSA 客户端验证中间件

### 功能介绍

`RSAClientAuthMiddleware` 是一个基于 RSA PSS 签名的客户端验证中间件，用于验证客户端的合法性。它能够：

- 从请求头获取客户端 ID、签名、随机数和时间戳
- 使用 RSA PSS 算法验证客户端签名的有效性
- 支持多种公钥格式的解析（X.509 和 OpenSSH）
- 验证通过后将客户端 ID 存入上下文

### 适用场景

适用于需要高安全性的 API 接口，特别是服务间通信、第三方应用集成以及需要防止重放攻击的场景。

### 使用方法

```go
import (
	"github.com/DotNetAge/sparrow/pkg/adapter/http/middlewares"
	"github.com/gin-gonic/gin"
)

func main() {
	// 创建 Gin 引擎
	router := gin.Default()
	
	// 配置客户端公钥映射表（客户端ID -> PEM格式公钥字符串）
	clientKeys := map[string]string{
		"client-123": "-----BEGIN PUBLIC KEY-----...-----END PUBLIC KEY-----",
		"client-456": "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQ...",
	}
	
	// 创建 RSA 客户端验证中间件
	middleware, err := middlewares.NewRSAClientAuthMiddleware(clientKeys)
	if err != nil {
		// 处理错误
	}
	
	// 应用中间件到需要客户端验证的路由组
	apiGroup := router.Group("/api")
	apiGroup.Use(middleware)
	{
		// 受保护的路由
		apiGroup.POST("/data/submit", submitData)
	}
	
	// 启动服务器
	router.Run(":8080")
}

// 在处理函数中获取客户端信息
submitData(c *gin.Context) {
	// 从上下文中获取客户端ID
	clientID, _ := c.Get("clientID")
	
	c.JSON(200, gin.H{
		"message": "数据提交成功",
		"clientID": clientID,
	})
}
```

### 客户端请求格式

客户端在发送请求时需要在请求头中包含以下字段：

- `X-Client-ID`: 客户端唯一标识符
- `X-Client-Signature`: 使用客户端私钥对 `X-Client-ID + X-Nonce + X-Timestamp` 进行 RSA PSS 签名后，再进行 Base64 编码的结果
- `X-Nonce`: 随机字符串，用于防止重放攻击
- `X-Timestamp`: 时间戳，用于防止重放攻击

### 错误处理

中间件会在不同的错误情况下返回相应的状态码和错误消息：

- 缺少必要的请求头：返回 401 状态码和 "缺少必要的客户端验证信息" 错误
- 未授权的客户端：返回 401 状态码和 "未授权的客户端" 错误
- 无效的签名格式：返回 401 状态码和 "无效的签名格式" 错误
- 签名验证失败：返回 401 状态码和 "客户端签名验证失败" 错误

## 中间件组合使用

在实际应用中，您可以根据需要组合使用这些中间件，以实现更全面的安全控制：

```go
import (
	"github.com/DotNetAge/sparrow/pkg/adapter/http/middlewares"
	"github.com/casbin/casbin/v2"
	"github.com/gin-gonic/gin"
)

func main() {
	// 创建 Gin 引擎
	router := gin.Default()
	
	// 配置 JWT 密钥
	jwtSecret := []byte("your-secret-key-here")
	
	// 初始化 Casbin 授权器
	enforcer, _ := casbin.NewEnforcer("path/to/model.conf", "path/to/policy.csv")
	
	// 配置客户端公钥
	clientKeys := map[string]string{
		"service-a": "-----BEGIN PUBLIC KEY-----...-----END PUBLIC KEY-----",
	}
	clientAuthMiddleware, _ := middlewares.NewRSAClientAuthMiddleware(clientKeys)
	
	// 公开路由（无需认证和授权）
	router.GET("/public/info", getPublicInfo)
	
	// 仅需要客户端验证的路由
	serviceGroup := router.Group("/service")
	serviceGroup.Use(clientAuthMiddleware)
	{
		serviceGroup.POST("/sync", syncData)
	}
	
	// 需要用户认证的路由
	userGroup := router.Group("/user")
	userGroup.Use(middlewares.JWTAuthMiddleware(jwtSecret))
	{
		userGroup.GET("/profile", getUserProfile)
	}
	
	// 需要用户认证和权限控制的路由
	adminGroup := router.Group("/admin")
	adminGroup.Use(middlewares.JWTAuthMiddleware(jwtSecret))
	adminGroup.Use(middlewares.RBACMiddleware(enforcer))
	{
		adminGroup.GET("/users", listAllUsers)
		adminGroup.POST("/users/create", createUser)
	}
	
	// 启动服务器
	router.Run(":8080")
}
```

## 最佳实践

1. **中间件顺序**：在组合使用多个中间件时，建议按照以下顺序应用：客户端验证 -> 用户认证 -> 权限控制

2. **密钥管理**：妥善管理 JWT 密钥和 RSA 密钥对，避免硬编码在代码中，建议通过环境变量或配置文件加载

3. **错误处理**：为不同类型的错误提供明确的错误消息，便于客户端理解和处理

4. **性能优化**：对于高并发场景，可以考虑缓存权限检查结果，减少重复计算

5. **安全性增强**：结合 HTTPS 使用这些中间件，确保传输过程中的数据安全

通过合理使用这些中间件，您可以为您的应用提供全面的安全保障，同时保持代码的简洁和可维护性。
