简介​

本文件为 Sparrow 应用框架的综合文档，面向两类读者：

  • 框架使用者：提供应用启动流程、配置系统、依赖注入容器、服务生命周期管理、数据库连接管理等的实用指南与最佳实践。
  • 框架扩展者：提供依赖注入容器的实现机制、配置加载与验证、扩展点设计与接入方式的深入实现细节。

项目结构​

Sparrow 采用分层与模块化组织方式，核心入口与基础设施集中在 bootstrap 包，配置系统集中在 config 包，业务能力通过选项函数（Option）进行装配与扩展。

  • 应用主控 App：负责应用生命周期、HTTP 服务器启动、优雅关闭、子进程与订阅器的管理、事件总线与消息流的接入。
  • IoC 容器 Container：支持匿名与具名依赖注册与解析，具备实例缓存与依赖自动解析能力。
  • 配置系统 Config/Loader：统一加载 YAML/JSON/TOML/Properties 等格式配置，支持 .env 环境变量覆盖，内置默认值与基础校验。
  • 数据库连接聚合 Database：聚合 Redis、SQL、Badger 等连接，便于容器统一注册与解析。
  • 授权与中间件 Authorization：集中管理 JWT/RSA/Casbin 等鉴权中间件，支持命名访问与顺序组合。

Sparrow 的启动与运行由“配置加载 → 容器注册 → 应用装配 → 服务启动 → 优雅关闭”构成的闭环。

应用启动流程​

  • 配置加载：优先从应用名对应的配置文件加载，支持多格式探测；同时加载 .env 并自动映射环境变量；设置默认值与基础校验。
  • 日志与路由：基于配置创建日志器与 Gin 引擎。
  • 容器初始化：创建 IoC 容器，随后通过 Option 注入数据库、消息总线、任务调度、会话、鉴权中间件等。
  • 子进程与重试：支持 Startable 子进程的启动与指数退避重试；失败项进入后台重试队列。
  • HTTP 服务：根据配置启动 HTTP 服务器；监听系统信号，执行优雅关闭与资源清理。
  • 配置模型：Config 聚合各子系统配置（应用、服务器、CORS、日志、NATS、SQL、Redis、Badger、RabbitMQ、Kafka、Casbin）。
  • 默认值：SetDefaults 为各字段设置合理默认值，确保最小可用配置。
  • 加载策略：支持多格式配置文件与 .env 覆盖；自动映射环境变量键名（点号转下划线）。
  • 校验：validateConfig 对必要字段进行基础校验（如端口范围、应用名非空）。
  • 注册：Register/ RegisterNamed 支持匿名与具名提供者；构造函数需至少一个返回值。
  • 解析：Resolve 支持具名与匿名解析；内部通过反射解析构造函数参数并递归解析依赖；首次解析结果会被缓存。
  • 辅助：ResolveInstance（匿名）与 ResolveByName（具名）简化调用；错误处理明确（类型不匹配、依赖缺失等）。
  • 子进程与订阅器：通过 NeedCleanup/GracefulClose 接口统一纳入 App.CleanUp 管理；支持超时控制与错误记录。
  • 重试机制：子进程启动失败时进入后台重试队列，采用指数退避（最多1分钟，最大重试10次），提升启动鲁棒性。
  • 优雅关闭：监听系统信号，提供30秒超时优雅关闭 HTTP 服务器，并清理所有子进程与订阅器。
  • 服务器：ServerHost/ServerPort 快速修改主机与端口。
  • 数据库：SQLiteDB/RedisDB/BadgerDB 提供不同存储后端的连接注册。
  • 消息与事件：Nats/NatStreamBus 提供 NATS 连接与事件总线；WithProjection 注册全量投影。
  • 任务系统：Tasks 提供调度器包装；Sessions 注册会话服务与路由。
  • 中间件：Middlewares 直接挂载；WithJWT/WithRSA/WithRBAC 注册鉴权中间件并注入 Auth。
  • 健康检查：HealthCheck 注册健康检查路由。
  • 调试模式：DebugMode 控制容器行为（如 Badger 内存模式）。

  • Database 聚合 Redis、SQL、Badger 连接，便于统一注册与解析。

  • 选项函数：

    • SQLiteDB：打开 sqlite3 连接，启用 WAL、同步策略、缓存大小、超时与外键约束，并执行 Schema。
    • RedisDB：基于配置创建 Redis 客户端。
    • BadgerDB：支持内存模式（调试）与生产配置（文件存储、压缩、阈值、内存表数量等）。

  • 仓库注册：BadgerRepo/SQLRepo/RedisRepo/MemRepo 为实体注册具名仓库，命名规则与前缀生成由工具函数确定。

  • 配置文件格式：YAML/YML/JSON/TOML/Properties/Props/Prop，按顺序探测首个存在的文件。

  • 环境变量：.env 文件加载后，通过 AutomaticEnv 自动映射；点号键名转换为下划线（如 app.name → APP_NAME）。

  • 默认值：SetDefaults 为各子系统提供默认值，减少手工配置。

  • 校验：validateConfig 对应用名与端口进行基础校验。

  • Authorization：集中管理 TokenGenerator 与中间件列表；支持命名中间件注册与获取。

  • WithJWT：注册 Token 生成器与 JWT 中间件。

  • WithRSA：从 KeyPath 加载客户端公钥，注册 RSA 客户端认证中间件。

  • WithRBAC：注册 Casbin Enforcer 并注入 RBAC 中间件。

  • 中间件挂载：Middlewares 直接挂载到 Gin 引擎。

  • App 依赖 Config、Container、Logger、Gin Engine、Authorization、Scheduler、Subscribers 等。

  • Container 依赖反射解析构造函数参数，形成依赖链。

  • Loader 与 Config 共同完成配置加载与校验。

  • 选项函数通过 Register/ RegisterNamed 将外部依赖注入容器。

  • 容器解析：依赖链解析与实例缓存显著降低重复创建成本；建议将高频使用的单例服务注册为具名，便于复用。
  • 数据库：SQLite 启用 WAL 与同步策略提升并发；Redis/SQL/Badger 的连接池与超时配置需结合负载评估调整。
  • NATS/JetStream：事件流 MaxAge 与压缩策略影响存储与传输开销，应按业务保留策略配置。
  • 重试策略：指数退避上限与最大重试次数可避免雪崩效应，建议结合监控告警。

故障排查指南​

  • 配置加载失败：确认配置文件格式与路径；检查 .env 是否存在且键名映射正确；查看默认值是否满足最小需求。
  • 依赖解析失败：检查构造函数签名（至少一个返回值）；确认依赖是否已注册；具名解析与匿名解析共享缓存，注意命名一致性。
  • 数据库连接异常：核对驱动、DSN、凭据与网络连通性；SQLite Schema 执行失败需检查 SQL 语句与权限。
  • NATS 连接失败：核对 URL 与认证信息；确认 NATS 服务可达。
  • HTTP 服务器无法优雅关闭：检查子进程/订阅器 Close 实现与超时设置。

Sparrow 通过清晰的配置系统、灵活的 IoC 容器与可插拔的选项机制，提供了从启动到运行的完整基础设施。使用者可通过少量选项快速装配所需能力；扩展者可基于容器与选项体系无缝集成自定义服务与中间件。建议在生产环境中结合监控与日志，持续优化数据库与消息系统的参数配置。

附录​

配置项一览与默认值​

  • 应用与安全

    • app.name: 应用名称（默认：sparrow）
    • app.version: 版本号（默认：1.0.0）
    • app.secret: 密钥（默认：随机生成）
    • app.key_path: RSA 公钥目录（默认：./data/perms）

  • 服务器

    • server.host: 主机地址（默认：0.0.0.0）
    • server.port: 端口（默认：8080）

  • CORS

    • cors.allow_origins: 允许来源（默认：["*"]）
    • cors.allow_methods: 允许方法（默认：["GET","POST","PUT","DELETE","OPTIONS"]）
    • cors.allow_headers: 允许头（默认：["Origin","Content-Type","Accept","Authorization"]）
    • cors.allow_credentials: 允许凭证（默认：true）
    • cors.max_age_hours: 预检缓存小时（默认：1）

  • 日志

    • log.level: 日志级别（默认：info）
    • log.format: 输出格式（默认：json）
    • log.output: 输出位置（默认：stdout）
    • log.filename: 文件名（默认：./logs/info.log）
    • log.mode: 模式（默认：dev）

  • NATS

    • nats.url: 服务地址（默认：nats://localhost:4222）
    • nats.stream_name: 事件流名称（默认：default_stream）
    • nats.max_age: 事件流最大保存天数（默认：0）

  • Redis

    • redis.host: 主机（默认：localhost）
    • redis.port: 端口（默认：6379）
    • redis.password: 密码（默认：空）
    • redis.db: 数据库编号（默认：0）
    • redis.es_db: 事件存储数据库编号（默认：1）

  • Badger

    • badger.data_dir: 仓库存储目录（默认：./data/badger/views）
    • badger.es_dir: 事件存储目录（默认：./data/badger/es）
    • badger.value_threshold: 值阈值（默认：1MB）
    • badger.num_compactors: 并发压缩器数量（默认：1）
    • badger.in_memory: 内存模式（默认：false）

  • Casbin

    • casbin.model_path: 模型文件路径（默认：./data/casbin/model.conf）
    • casbin.policy_path: 策略文件路径（默认：./data/casbin/policy.csv）

  • SQL

    • sql.driver: 驱动（默认：postgres）
    • sql.host: 主机（默认：localhost）
    • sql.port: 端口（默认：5432）
    • sql.user: 用户（默认：postgres）
    • sql.password: 密码（默认：postgres）
    • sql.dbname: 数据库名（默认：postgres）
    • sql.es_dbname: 事件存储数据库名（默认：postgres）

  • 键名替换：将配置键中的点号（.）替换为下划线（_），例如 app.name → APP_NAME。

  • 自动映射：启用 AutomaticEnv 后，环境变量将自动覆盖配置文件中的同名键。

  • 添加自定义服务

    • 在选项函数中通过 Container.Register/ RegisterNamed 注册构造函数。
    • 若服务依赖其他服务，构造函数参数将由容器自动解析。

  • 添加自定义中间件

    • 使用 Middlewares 直接挂载到 Gin 引擎。
    • 或通过 Authorization.AddMiddleware 注册命名中间件并在路由中按需启用。

  • 添加自定义仓库

    • 使用 BadgerRepo/SQLRepo/RedisRepo/MemRepo 为实体注册具名仓库。
    • 命名规则：实体短名 + “Repo”，前缀使用实体短名的蛇形命名。

  • 添加自定义消息总线

    • 通过 NATS 连接注册自定义 Bus/Hub，并纳入 App.NeedCleanup 管理。
