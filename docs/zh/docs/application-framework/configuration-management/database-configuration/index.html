简介​

本文件面向数据库配置的综合说明，涵盖关系型数据库（PostgreSQL、MySQL、SQLite）、Redis 以及 BadgerDB 的配置要点、连接参数、驱动选择与连接池设置，并提供性能优化建议与故障排除指南。文档基于仓库中的配置模型与启动流程进行整理，帮助开发者快速理解与部署。

项目结构​

数据库配置主要分布在以下模块：

  • 配置层：定义各类数据库配置结构体与默认值加载
  • 启动层：根据配置创建数据库连接与仓储实例
  • 仓储层：提供通用的 CRUD 与查询能力封装
  • 配置结构体
    • SQLConfig：驱动、主机、端口、用户名、密码、数据库名、事件存储专用库名
    • RedisConfig：主机、端口、密码、数据库编号、事件存储专用数据库编号
    • BadgerConfig：数据目录、事件存储目录、值阈值、压缩器、紧凑器数量
  • 默认值与加载
    • SetDefaults：为各配置项设置默认值（如 Redis/SQL/Badger 的默认主机、端口、路径等）
    • Load：加载配置文件、环境变量、校验配置有效性
  • 启动与连接
    • Database：统一持有 Redis、SQL、Badger 客户端
    • 启动选项：注册 SQL/Redis/Badger 实例，构建仓储

数据库配置与启动的整体流程如下：

关系型数据库配置（PostgreSQL、MySQL、SQLite）​

  • 驱动与连接参数
    • 驱动：通过 SQLConfig.Driver 指定（默认 postgres）
    • 连接参数：Host、Port、User、Password、Dbname、ESDbname（事件存储专用库名）
    • DSN 构造：SQLConfig.Dsn()/ESDsn() 提供标准 DSN 格式
  • 连接池设置
    • 代码中未直接暴露连接池参数（如最大连接数、空闲数、生命周期），但可通过数据库驱动的扩展接口在启动阶段进行配置
    • 示例：SQLiteDB 启动选项中对 sqlite3 进行了 PRAGMA 配置（WAL、同步级别、缓存大小、busy_timeout、外键约束）
  • 事件存储专用库
    • ESDsn 与 ESDbname 用于事件存储场景下的独立数据库
  • 连接信息与认证
    • 主机与端口：Host、Port
    • 密码认证：Password
    • 数据库选择：DB（默认 0）
    • 事件存储专用数据库：ESDB（默认 1）
  • 客户端创建
    • newRedis 根据配置构造 redis.Client，包含 Addr、Password、DB
  • 数据目录与事件存储目录
    • DataDir：普通视图存储目录
    • EventStoreDir：事件存储目录
  • 性能参数
    • ValueThreshold：阈值，决定小值是否进入 LSM 树
    • NumCompactors：紧凑器数量
    • 压缩器：默认 ZSTD（可在启动选项中设置）
  • 内存与文件大小
    • 启动选项中设置了内存表大小、值日志文件大小等参数，用于平衡吞吐与资源占用
  • SQL 仓储
    • 泛型实现，支持事务、软删除检测、分页、条件查询、批量操作
    • 通过底层 sql.DB 执行 SQL 操作
  • Redis 仓储
    • 基于键前缀命名空间，支持 TTL、批量 Pipeline、全表扫描条件查询
  • Badger 仓储
    • 基于键前缀与迭代器，支持事务读写、批量更新、分页与条件过滤
  • 配置到启动的依赖
    • Config 聚合 SQL/Redis/Badger 配置
    • Load 负责默认值注入与校验
    • 启动选项根据配置创建具体客户端并注册到容器
  • 仓储到数据库的依赖
    • 仓储通过底层数据库客户端执行操作
    • 不同数据库的特性（如软删除、事务）在仓储层体现
  • 关系型数据库（PostgreSQL/MySQL/SQLite）
    • 连接池参数未在配置中直接暴露，可在启动阶段通过驱动扩展接口设置（如 sqlite3 的 PRAGMA 已示例）
    • 建议结合业务并发与查询特征调整最大连接数、空闲连接数、连接生命周期
  • Redis
    • 使用 Pipeline 批量操作减少 RTT
    • 合理设置 TTL 与键空间前缀，避免全表扫描
    • 根据数据规模选择合适的数据库编号，隔离事件存储
  • BadgerDB
    • 通过启动选项设置内存表大小、值日志文件大小与压缩器，平衡吞吐与资源
    • ValueThreshold 控制小值与大值的存储策略，影响 LSM 树结构与读写性能
    • NumCompactors 影响后台压缩负载，需按数据增长速率调优

[本节为通用性能建议，不直接分析具体文件]

故障排除指南​

  • 配置加载失败
    • 检查配置文件格式与路径，确认默认值是否正确设置
    • 确认环境变量键名替换规则与实际环境变量一致
  • 连接失败
    • 校验主机、端口、密码、数据库名是否正确
    • 对 Redis：确认密码与数据库编号；对 SQL：确认 DSN 格式与驱动可用性
  • 运行时错误
    • SQL：关注事务开启、回滚与提交时机，确保实体 ID 与软删除字段存在
    • Redis：注意键前缀与 TTL 设置，避免误删或过期导致的数据丢失
    • Badger：注意键前缀与迭代器使用，避免并发读写冲突

本项目提供了清晰的数据库配置模型与启动流程，支持关系型数据库、Redis 与 BadgerDB 的基础配置与运行时访问。对于连接池与更细粒度的性能参数，可在启动阶段通过驱动扩展接口进行补充配置。建议结合业务场景对连接池、压缩策略与键空间设计进行针对性优化，并在生产环境中加强配置校验与监控告警。
