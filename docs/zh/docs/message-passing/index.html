引言​

本文件面向消息系统开发者，系统性梳理 Sparrow 消息传递体系：事件总线架构、NATS 集成、发布订阅模式、序列化与路由策略、可靠性保障、以及对 RabbitMQ、Redis 等协议的支持现状与扩展建议。文档同时覆盖消息过滤、去重与幂等性实践、批量与流式处理等高级特性，并提供可视化图示与最佳实践。

项目结构​

消息传递相关代码主要分布在以下模块：

  • 消息总线抽象与实现：messaging 包中的 bus、core、publisher、subscriber、jspub、jssub、nats_reader 等
  • 事件总线（EventBus）：eventbus 包下的内存、NATS、RabbitMQ、Redis 实现
  • 配置：config 包下的 NATsConfig、RabbitMQConfig、RedisConfig
  • 接口与约束
    • StreamPublisher：定义发布能力（单条与批量）
    • StreamSubscriber：定义订阅能力（Start/Stop）
    • StreamReader：定义事件读取与重放能力（GetEvents/Replay/ReplayFromVersion/ReplayFromOffset）
    • DomainEventConstraint：泛型约束，限定为实现 entity.DomainEvent 的类型
  • 事件总线适配
    • EventPublisher：将事件存储中的领域事件发布到通用事件总线（含主题命名与序列化）
    • EventSubscriber[T]：订阅通用事件总线，按“服务名.聚合类型.事件类型”主题过滤并反序列化为具体领域事件
  • NATS 事件流
    • JetStreamPublisher：基于 JetStream 的事件发布，自动确保流存在，主题为“聚合类型.聚合ID.事件类型”
    • JetStreamSubscriber[T]：基于 JetStream 的事件订阅，显式确认、优雅关闭、临时消费者模式
    • JetStreamReader：基于 JetStream 的事件读取与重放，支持按版本与偏移量重放
  • 总线 Hub
    • StreamHub：封装订阅器集合，提供 AddSub、Start、Close 等统一入口

Sparrow 的消息传递体系分为三层：

  • 事件总线层：通用事件总线（内存/NATS/RabbitMQ/Redis），用于跨服务广播与订阅
  • 事件流层：基于 NATS JetStream 的事件流，用于事件存储与重放
  • 应用集成层：StreamHub/EventPublisher/EventSubscriber 统一接入与编排

事件总线适配：EventPublisher 与 EventSubscriber​

  • 主题命名与序列化
    • EventPublisher 将领域事件序列化为 JSON 并 Base64 编码，构造通用事件，主题格式为“服务名.聚合类型.事件类型”，发布到通用事件总线
    • EventSubscriber 通过主题过滤（支持服务名、聚合类型、事件类型三段式），对 Payload 进行 Base64 解码与 JSON 反序列化，再调用具体处理器
  • 处理器注册与并发
    • EventSubscriber 使用 map 存储处理器，加读写锁保护；支持按服务名过滤与通配符聚合类型匹配
  • 错误处理
    • 对 Base64 解码、JSON 反序列化、订阅注册失败等情况记录日志并返回错误
  • 发布（JetStreamPublisher）
    • 自动确保流存在（主题集合为“聚合类型.>”），发布主题为“聚合类型.聚合ID.事件类型”
    • 提供选项：最大消息年龄、最大流大小、最大消息大小、调试模式（内存存储）
  • 订阅（JetStreamSubscriber[T]）
    • 显式确认策略，持久化消费者（Durable），支持从最早消息开始投递与即时重放
    • 优雅关闭：通过 context 取消、WaitGroup 等待处理完成、清理资源
    • 消息处理：反序列化 BaseEvent，再由实体工具反序列化为具体事件类型，调用处理器，失败时 NAK
  • 读取（JetStreamReader）
    • 临时消费者（非持久化），支持按聚合 ID 过滤、版本过滤、偏移量过滤
    • 全量拉取与排序后应用到聚合根，便于重放与投影重建
  • StreamHub 将订阅器集合封装为 Subscribers 接口，提供 AddSub、Start、Close 等统一入口，便于应用侧集中管理
  • 与 NewJetStreamBus/StreamPub/StreamReader 等组件协同工作
  • 组件耦合
    • JetStreamPublisher/JetStreamSubscriber/JetStreamReader 依赖 NATS JetStream 客户端
    • EventPublisher/EventSubscriber 依赖通用事件总线接口与实体序列化工具
    • StreamHub 依赖 Subscribers 接口，便于替换不同实现（内存/事件流/NATS 等）
  • 外部依赖
    • NATS（JetStream）、RabbitMQ、Redis 的事件总线实现分别位于 eventbus 下的子目录
    • 配置项通过 config 包注入（NATS/RabbitMQ/Redis）

  • 批量发布

    • JetStreamPublisher/Publisher 均提供 PublishEvents 批量接口，减少网络往返与系统调用开销

  • 流式读取与重放

    • JetStreamReader 使用 FetchNoWait 分批拉取，结合排序与按版本/偏移量过滤，降低一次性内存占用

  • 消费者生命周期

    • JetStreamSubscriber[T] 通过 WaitGroup 与 context 控制优雅关闭，避免消息丢失与资源泄漏

  • 资源清理

    • JetStreamReader 使用临时消费者，消费完成后自动清理，降低控制面压力

  • 发布失败

    • JetStreamPublisher：检查流是否存在、主题是否匹配、消息大小限制、序列化错误
    • EventPublisher：检查服务名、主题拼接、JSON 序列化与 Base64 编码

  • 订阅失败

    • JetStreamSubscriber[T]：确认消费者创建、过滤主题、显式确认、优雅关闭流程
    • EventSubscriber[T]：确认主题过滤条件、Base64 解码与反序列化链路

  • 读取异常

    • JetStreamReader：关注流不存在、临时消费者创建、批次拉取与消息确认

  • 日志定位

    • 各组件均输出关键字段（如 consumer、subject、stream、error），便于快速定位问题

Sparrow 的消息传递体系以统一接口抽象事件总线与事件流，结合 NATS JetStream 实现高可靠、可重放的事件存储与订阅模型。通过清晰的主题命名、严格的序列化与反序列化流程、显式确认与优雅关闭机制，系统在可维护性与可扩展性上具备良好基础。对于 RabbitMQ、Redis 等协议，可通过 eventbus 子包扩展实现，以满足不同部署与性能需求。

消息过滤、去重与幂等性​

  • 过滤

    • 事件总线：EventSubscriber[T] 支持服务名、聚合类型、事件类型三段式过滤
    • 事件流：JetStreamSubscriber[T] 使用 FilterSubject 精确订阅

  • 去重

    • 建议在应用层基于事件 ID 或聚合版本进行去重（例如在处理器中维护已处理事件集合）

  • 幂等性

    • 建议在处理器中实现幂等逻辑（如按事件 ID 去重、按版本号判断是否已应用）

  • 批量发布：JetStreamPublisher/Publisher 的 PublishEvents

  • 流式读取：JetStreamReader 的 FetchNoWait 分批拉取与排序

  • 已有实现

    • 事件总线：内存、NATS、RabbitMQ、Redis
    • 事件流：NATS JetStream

  • 扩展建议

    • 新增协议：在 eventbus 下新增子包，实现 EventBus 接口；在 messaging 中提供相应 Publisher/Subscriber/Reader
    • 配置：在 config 包中新增对应配置结构体，并在引导阶段注入
