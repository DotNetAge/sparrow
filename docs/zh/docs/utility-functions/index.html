简介​

本文件为 Sparrow 工具函数库的实用参考文档，聚焦以下能力：

  • ID 生成工具：UUID 生成与雪花 ID 生成器
  • 类型转换工具：字符串、整数、时间、命名风格转换与复数化
  • 排序工具：泛型实体排序、稳定排序、按 JSON 标签字段排序、随机打乱
  • 时间处理工具：时间解析与命名风格转换
  • 类型名工具：泛型类型名与短类型名提取
  • 扩展与最佳实践：如何在项目中正确使用这些工具，避免常见误区，并提供扩展与自定义开发指南

项目结构​

工具函数主要位于 pkg/utils 目录，配合 usecase 中的 SortField 结构体进行排序控制；部分仓库层会调用排序工具；另有 codes 目录提供雪花 ID 与编码生成器作为 ID 生成的补充方案。

  • ID 生成工具
    • UUID 生成：提供全局唯一字符串 ID
    • 雪花 ID 生成：基于时间戳、机器 ID、序列号的高吞吐分布式 ID
    • 编码生成：前缀+日期+随机码式编码
  • 类型转换工具
    • 字符串/整数/布尔/浮点/指针/结构体等通用转换
    • 时间解析：RFC3339、常见本地格式、Unix 秒时间戳
    • 命名风格转换：下划线、帕斯卡、驼峰、kebab、复数
  • 排序工具
    • 泛型实体排序：支持默认按创建时间降序、按字段升/降序
    • 反射字段访问：支持直接字段名与 JSON 标签映射
    • 随机打乱：Fisher-Yates 洗牌
  • 时间处理工具
    • ParseTime：统一解析多种时间输入
    • 命名风格转换：与数据库/前端命名约定对齐
  • 类型名工具
    • GetTypeName/GetShotTypeName：获取泛型类型全名与短名

工具函数在系统中的位置与交互如下：

ID 生成工具​

  • UUID 生成

    • 功能：生成全局唯一字符串 ID
    • 参数：无
    • 返回：字符串 ID
    • 使用示例路径：生成示例
    • 性能特点：依赖第三方库，生成开销低，适合大多数场景
    • 适用场景：通用主键、临时标识、日志追踪 ID

  • 雪花 ID 生成器

    • 功能：高吞吐分布式 ID，包含时间戳、机器 ID、序列号
    • 参数：机器 ID（0-1023）、可选起始时间戳
    • 返回：64 位整数 ID
    • 使用示例路径：构造与生成
    • 性能特点：单机高吞吐，避免跨机时钟回拨；同一毫秒内序列号上限 4095
    • 适用场景：高并发写入、强一致顺序需求

  • 编码生成器

    • 功能：前缀+日期+随机码式编码
    • 参数：业务前缀
    • 返回：字符串编码
    • 使用示例路径：编码生成
    • 性能特点：简单易用，便于人类可读
    • 适用场景：订单号、任务号等业务编码

  • ToString

    • 功能：将任意类型转为字符串
    • 参数：任意接口
    • 返回：字符串
    • 使用示例路径：字符串转换

  • ToInt

    • 功能：将任意类型转为 int
    • 参数：任意接口
    • 返回：int
    • 使用示例路径：整数转换

  • ParseTime

    • 功能：解析多种时间输入为 time.Time
    • 参数：字符串、RFC3339、本地格式、Unix 秒时间戳
    • 返回：time.Time
    • 使用示例路径：时间解析

  • 命名风格转换

    • Camel/Pascal/Snake/Kebab：下划线、帕斯卡、驼峰、kebab 互转
    • 参数：字符串
    • 返回：转换后的字符串
    • 使用示例路径：命名转换

  • 复数化

    • 功能：简单规则的英文复数转换
    • 参数：字符串
    • 返回：复数形式
    • 使用示例路径：复数化

  • SortEntities

    • 功能：对实体切片进行稳定排序
    • 参数：
      • entities：任意泛型切片
      • sortFields：排序字段列表（usecase.SortField）
      • defaultSortByCreatedAt：是否默认按创建时间降序
    • 返回：无（原地修改）
    • 使用示例路径：排序测试
    • 算法实现要点：
      • 默认排序：若实体实现创建时间接口，则按时间降序
      • 字段排序：优先比较各字段，支持字符串/整数/浮点/布尔/时间
      • 字段定位：优先按字段名，其次按 JSON 标签（tag）匹配
    • 性能特点：稳定排序 O(n log n)，默认路径避免反射，字段排序路径使用反射
    • 使用建议：
      • 为实体实现创建时间接口可获得更高效的默认排序
      • 字段排序时确保字段存在且类型可比较

  • compareEntities

    • 功能：通过反射比较两个实体的指定字段
    • 使用示例路径：字段比较

  • getFieldByJSONTag

    • 功能：通过 JSON 标签查找字段
    • 使用示例路径：JSON 标签查找

  • compareValues

    • 功能：根据字段类型进行比较
    • 使用示例路径：值比较

  • ShuffleStrings

    • 功能：Fisher-Yates 洗牌
    • 参数：字符串切片
    • 返回：无（原地修改）
    • 使用示例路径：洗牌

  • ParseTime

    • 支持输入：RFC3339、本地格式、Unix 秒时间戳
    • 返回：time.Time；解析失败返回零值
    • 使用示例路径：时间解析

  • 命名风格转换

    • 与数据库/前端命名约定对齐，便于序列化/反序列化
    • 使用示例路径：命名转换

  • GetTypeName

    • 功能：获取泛型类型全名
    • 参数：无（泛型）
    • 返回：字符串
    • 使用示例路径：类型名获取

  • GetShotTypeName

    • 功能：获取短类型名（去除包路径）
    • 参数：无（泛型）
    • 返回：字符串
    • 使用示例路径：短类型名获取

  • SortEntities 依赖 usecase.SortField 定义

  • 仓储层（如 RedisRepository）通过调用 utils.SortEntities 实现排序

  • 命名转换工具被多个仓库层用于字段命名规范化

  • ID 生成工具在多处被仓储层用于生成实体 ID

  • SortEntities
    • 默认路径避免反射，优先使用类型断言获取创建时间，复杂度 O(n log n)
    • 字段排序路径使用反射，注意字段数量与实体规模
  • ShuffleStrings
    • Fisher-Yates 洗牌，时间复杂度 O(n)，空间复杂度 O(1)
  • ParseTime
    • 多格式尝试解析，注意输入格式一致性以减少失败重试
  • 命名转换
    • 字符串拼接与切分，注意在高频场景下缓存中间结果

故障排查指南​

  • 排序无效或异常
    • 确认实体实现了创建时间接口（若依赖默认排序）
    • 确认排序字段名与实体字段或 JSON 标签一致
    • 参考测试用例验证行为：排序测试
  • 命名转换结果不符合预期
    • 检查输入字符串是否包含大小写混合与数字
    • 参考转换函数实现：命名转换
  • 时间解析失败
    • 确认输入格式符合 RFC3339 或本地格式
    • 参考解析逻辑：时间解析
  • 类型名工具异常
    • 确认传入的是具体类型而非接口类型
    • 参考测试用例：类型名测试

Sparrow 工具函数库提供了覆盖 ID 生成、类型转换、排序、时间处理与类型名提取的实用工具集。通过泛型与稳定排序策略，结合命名规范与时间解析能力，能够满足多数工程场景的需求。建议在项目中遵循“默认路径优先、字段排序谨慎、命名风格统一”的原则，并在高并发场景选择合适的 ID 生成方案。
