简介​

任务包装器是 Sparrow 任务管理系统的核心抽象层，它为任务的提交、执行、监控和管理提供了统一的接口。该设计采用了装饰器模式和拦截器模式，通过包装器链的形式实现了横切关注点的模块化处理。

任务包装器的主要设计目标包括：

  • 统一接口：为不同类型的调度器提供一致的 API
  • 装饰器模式：支持任务执行过程中的横切关注点处理
  • 拦截器模式：在任务执行前后插入自定义逻辑
  • 可扩展性：支持自定义包装器的组合和配置
  • 监控能力：内置任务状态监控、日志记录和性能统计

项目结构​

任务包装器 (SchedulerWrapper)​

任务包装器是整个任务系统的核心控制器，负责：

  • 任务提交：提供多种任务提交方式（立即执行、定时执行、周期性执行）
  • 配置管理：支持重试次数、并发数、执行模式等配置
  • 调度器代理：将任务委托给底层调度器执行
  • 状态监控：提供任务状态查询和列表功能

任务构建器采用建造者模式，提供了灵活的任务配置能力：

  • 链式配置：支持连续的配置调用
  • 参数验证：在构建时验证任务参数的有效性
  • 默认值设置：为缺失的参数设置合理的默认值
  • 类型安全：确保任务配置的类型安全性

调度器接口定义了任务系统的核心行为：

  • 任务生命周期管理：调度、取消、状态查询
  • 并发控制：支持最大并发任务数的配置
  • 优雅关闭：提供带超时控制的关闭机制
  • 扩展性：支持不同执行模式的调度器实现

内存管理优化​

任务包装器实现了智能的内存管理策略：

  • 任务数量限制：默认最多保存 500 个任务
  • TTL 机制：已完成任务 5 分钟后自动清理
  • LRU 清理：删除最老的已完成任务以腾出空间

并发性能调优​

  • 并发槽位控制：通过条件变量精确控制并发任务数量
  • 等待策略：使用轮询和条件变量结合的方式避免阻塞
  • 超时控制：支持任务级别的超时控制

执行效率优化​

  • 任务类型缓存：混合调度器缓存任务类型到执行模式的映射
  • 状态快速查询：直接访问任务状态而无需复杂查询
  • 批量操作支持：支持批量任务状态查询和管理

故障排除指南​

常见问题诊断​

  1. 任务无法执行

    • 检查任务处理器是否为空
    • 验证任务调度时间是否有效
    • 确认调度器是否已启动

  2. 并发控制失效

    • 检查最大并发数配置
    • 验证并发槽位是否被正确释放
    • 确认等待通道是否正常工作

  3. 重试机制异常

    • 检查重试次数配置
    • 验证重试延迟计算
    • 确认任务状态转换逻辑

调试技巧​

  • 使用 ListTasks() 方法查看所有任务状态
  • 通过 GetTaskStatus() 获取特定任务详情
  • 监控活跃任务数和最大并发数
  • 检查任务日志和错误信息

任务包装器通过精心设计的架构实现了以下目标：

  1. 统一抽象：为不同类型的调度器提供一致的 API 接口
  2. 横切关注点：通过装饰器模式实现监控、日志、重试等功能的模块化
  3. 可扩展性：支持自定义调度器和包装器的组合
  4. 性能优化：实现了高效的并发控制和内存管理
  5. 可靠性保障：内置重试机制和错误处理

该设计为 Sparrow 任务系统提供了强大的基础，支持从简单任务到复杂业务场景的各种需求。

配置选项详解​

选项类型默认值描述
WithRetryCountint3默认重试次数
WithSchedulerTaskSchedulerNewHybridScheduler()自定义调度器实例
WithMaxConcurrentint无限制最大并发任务数
WithConcurrentstring...无指定任务类型为并发执行
WithSequentialstring...无指定任务类型为顺序执行

使用示例​

// 基础使用
wrapper := NewSchedulerWrapper()

// 配置重试和并发
wrapper = NewSchedulerWrapper(
    WithRetryCount(3),
    WithMaxConcurrent(10),
)

// 指定任务类型执行模式
wrapper = NewSchedulerWrapper(
    WithSequential("critical-task"),
    WithConcurrent("batch-task"),
)
